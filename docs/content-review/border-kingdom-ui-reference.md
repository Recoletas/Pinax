# 边境王国 · 雾潮暮湾 UI 参考

## 用法

这不是让产品去模仿别人的皮肤，而是借 3 个成熟参考，收紧“单旗舰世界入口”的信息层级和决策手感。

目标只看四件事：

- 首屏先让用户看到什么
- 三个主要动作怎么排
- 势力 / 事件 / 任务怎么压成可扫读摘要
- 视觉氛围怎么服务“进入世界”，而不是做功能货架

## Reference 1: Citizen Sleeper

来源：

- https://store.steampowered.com/app/1578650/Citizen_Sleeper/

适合借的不是科幻风格，而是它的“先给压力，再给行动”。

### 可以借什么

- 首屏不是一堆系统说明，而是当前周期里最重要的几个行动位。
- 世界状态、人物状态、倒计时压力都压成很短的摘要块。
- 用户看到的是“我现在能做什么”，不是“这个系统能做什么”。

### 落到 Pinax 的做法

- 顶部先放 `开场困境`，不要先放长介绍。
- 中间只放 3 个行动钩子，而且每个钩子都带一个不同的风险或回报提示。
- 侧边或下方再放 `势力 / 地点 / 事件 / 任务` 的摘要，不要和主 CTA 抢视觉优先级。

### 文案和动线建议

- 主按钮文案要像行动，不要像目录：
  - `先去钟楼查痕迹`
  - `夜访码头核夜账`
  - `找证人问雾军`
- 不要写成：
  - `查看详情`
  - `进入设定`
  - `开始导入`

## Reference 2: Suzerain

来源：

- https://store.steampowered.com/app/1207650/Suzerain/

它适合借的是“把抽象政治压力变成可读局势面板”的方法。

### 可以借什么

- 复杂世界不是靠长文一次讲完，而是拆成几块正在起作用的局势。
- 每次选择都隐含阵营后果，这种压力应该在 UI 上可见。
- 叙事和局势面板不是两张皮，而是互相解释。

### 落到 Pinax 的做法

- `边境王国` 入口右侧可以用很轻的摘要区，显示：
  - `王室：黎明前要答案`
  - `行会：要求先封账`
  - `难民：掌握第一手证词`
- 让玩家在点击前就知道三条线分别会得罪谁、依赖谁。
- “更多题材”不要和旗舰世界并列，而应退到次级入口。

### 文案和动线建议

- 把势力摘要写成正在发生的压力句，而不是静态名词卡：
  - `巡骑团要求接管调查`
  - `行会试图封存夜账`
  - `学院拒绝公开完整观测记录`

## Reference 3: 80 Days

来源：

- https://store.steampowered.com/app/381780/80_Days/

它适合借的是“把一个很大的世界，压成几个立刻可选的路线”。

### 可以借什么

- 用户不需要在首屏知道整个世界，只需要知道接下来有哪 2-4 条有差异的路。
- 每个选择都既像剧情按钮，也像资源 / 风险判断。
- 次级路线存在，但不会和主路径抢焦点。

### 落到 Pinax 的做法

- `边境王国 · 雾潮暮湾` 应该是唯一主卡。
- 其余世界放进 `更多题材` 或 `归档`，做成短列表或折叠面板。
- 3 个行动钩子要直接表达玩法差异，而不是三个不同修辞的“开始”。

### 文案和动线建议

- 首屏结构建议：
  1. 世界名
  2. 开场困境
  3. 3 个行动钩子
  4. 势力 / 地点 / 事件摘要
  5. 次级入口：`更多题材`

## 不该借什么

- 不要把入口做成重卡片货架。
- 不要把世界信息塞成百科全书首屏。
- 不要让“预设导入 / 文本导入 / AI 生成”三个系统入口和旗舰世界同级抢首屏。
- 不要让 CTA 写成功能名词；要写成用户下一步要做的事。

## 对当前入口的直接约束

如果只保留一条结论，就是：

`首屏优先级必须是 开场困境 > 3 个行动钩子 > 势力压力摘要 > 其他世界 / 其他导入方式。`

这个顺序不稳，`边境王国` 就会重新退回“一个世界书预设”，而不是“今晚可以直接进去玩的世界”。
