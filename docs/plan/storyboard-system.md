# 06 - 编导模式方案

> 目标：把诗歌、散文、小说体验片段统一转换成分镜结构，便于导出和后续视频创作。

## 1. 当前问题

当前诗歌工坊和散文随笔已经有编导模式，但各自的数据结构和导出方式并不完全统一。

长期风险：

- 不同模块重复实现分镜结构。
- 导出 schema 不稳定。
- 同一作品不同部分难以复用。

## 2. 目标

统一成一个 storyboard schema。

```text
文本片段
  → 镜头结构
  → 版本化分镜
  → 导出
```

## 3. 统一 schema

建议字段：

- `shotId`
- `sourceText`
- `scene`
- `shotSize`
- `cameraMovement`
- `duration`
- `visual`
- `dialogue`
- `sound`
- `music`
- `transition`
- `emotion`
- `notes`

## 4. 具体方案

### 4.1 统一输入

编导模式的输入应支持：

- 诗歌
- 散文
- 小说片段
- 体验会话选段

### 4.2 版本化

每次生成分镜都保留版本，不覆盖旧版本。

版本至少包括：

- 版本号
- 生成时间
- 来源文本
- 生成参数

### 4.3 导出校验

导出前检查：

- 必填字段是否齐全
- 镜头顺序是否连续
- 时长是否有效
- 是否有空字段

## 5. 实施步骤

### 第一步

- 统一分镜 schema。
- 明确诗歌、散文、体验片段共用同一结构。

### 第二步

- 做版本化存储。
- 允许恢复旧版本。

### 第三步

- 增加导出校验。
- 导出失败时返回明确原因。

## 6. 验收标准

- 诗歌、散文、体验片段生成同一结构。
- 导出前有校验。
- 历史版本可恢复。
