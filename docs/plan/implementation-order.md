## 2. 当前推荐顺序

### 2.1 现在先完成阶段 E

目标：先把规划语言和剩余工作收束，不再用旧叙事带着实现走偏。

本阶段产出：

- 总计划重写。
- 当前执行计划重写。
- 专题页重写。
- 明确“素材是中转层，卡片画布是关系层，分镜是输出层”。
- 明确诗歌页没有独立保留必要性，原散文页不再保留题材定位。

退出条件：

- 文档体系不再把诗歌和散文写成两条并列产品主线，并完成诗歌退场规划。
- 剩余任务明确收束为编辑器与统一顾问、记忆层、分镜层和信息架构收口四条主线。

### 2.2 已完成阶段 F

目标：先解决写作主链路当前最重的体验问题，把高频自动补全改成更稳的统一智能顾问。

本阶段只做：

1. 决定编辑器路线，不上 ProseMirror。
2. 第一阶段保留 textarea，不做大编辑器迁移。
3. 删掉顾问 AI/LLM 模式，只保留 OpenClaw 统一顾问。
4. 把顾问能力改成自动收线、纠错、局部修复、章节体检。
5. 把自动补全降级为弱能力，默认不再高频自动触发。
6. OpenClaw 不可用时给明确错误和降级说明，不再用用户可见的后端切换兜底。

退出条件：

- 写作主链路不再以 ghost 续写作为核心智能交互。
- 顾问只有一个后端与一条调用链。
- 用户能对选区、段落、章节执行稳定的修正与收线动作。

### 2.3 已完成到可暂缓阶段 G

目标：把记忆系统从“可用”收口成“稳定可控”。

本阶段只做：

1. 候选批量整理。
2. 去重和冲突处理。
3. 同步状态和失败重试表达。

退出条件：

- 候选数量上来后仍可高效处理。
- 本地和远端记忆不会无提示重复或冲突。
- 用户能看见同步结果。

### 2.4 阶段 H 阶段性收口

目标：把分镜从页面功能升级成统一输出层。

本阶段顺序：

1. 先建统一 storyboard schema。（已启动）
2. 再建版本对象和校验器。（已启动）
3. 再按素材引用卡片、体验素材、章节片段的顺序接入来源；旧散文/诗歌只作为迁移输入。

其中旧散文、旧诗歌、体验素材和章节草稿的直接导出都已经接入统一文档，可作为迁移基础。后面优先把素材引用、关系画布和紧凑镜头预览做成主体验，复杂导出细节暂缓。

退出条件：

- 任一来源都能落成统一 storyboard 对象。
- 历史版本可恢复。
- 导出前有统一校验。

当前状态：

- 这条线已完成到可用基线，后续只补影响用户可用性的 UI 入口、校验反馈和导出提示。
- 不再把分镜细节扩展作为当前主线。

### 2.5 阶段 I 纳入当前 UI 主线

目标：把界面叙事和旧结构收口成真正的工作台模型。

本阶段包含：

- 导航命名调整。
- 入口重排。
- 原散文页改为卡片画布。
- 诗歌独立页迁出可复用生成动作后删除。
- 文案与 README 对外叙事统一。

当前已启动：

- 一级导航已改成体验、世界书、写作、素材、画布。
- 欢迎页与首页入口已开始同步收口。
- 素材页已统一到 `narrativeAssets` 资产池，写作页收件箱开始与素材页共享同一批资产。
- 素材页左栏已减负，素材可直接加入/定位卡片画布。
- 卡片节点已接入素材引用并压缩为关系预览节点，主入口不再指向诗歌页。

下一步：

- 收口画布到统一分镜版本的预览、校验和保存反馈。
- 迁移低成本可复用生成动作后，从路由中移除诗歌工作区并清理专属实现。

退出条件：

- 新用户能通过导航理解连续工作流。
- 用户理解的是素材、关系画布和分镜输出，而不是诗歌/散文页面。

### 2.6 当前推进阶段 J

目标：把已完成的底层能力整理成用户能直接理解和完成任务的 UI。

本阶段只按可用性排序：

1. 体验、世界书、写作、素材、分镜的核心路径先可完成。
2. 重复入口先合并或降级。
3. 固定浮层和移动端阻塞先修掉。
4. 视觉统一和动画细节放到后面。

退出条件：

- 用户不需要理解旧页面边界，也能完成体验到素材、素材到写作、设定导入、分镜保存这几条路径。
- 单页面不再堆叠多个互相竞争的主入口。
- 兼容检查不再暴露遮挡、滚动锁死或关键按钮不可点击。

## 3. 每阶段共同门槛

无论推进哪一阶段，都必须同时满足下面四个门槛：

1. 数据归属清楚。
任何新增字段和对象都必须能回答“它属于 session、素材、世界书、记忆还是 storyboard”。

2. 生成链路可追踪。
凡是主模型调用都必须进入统一任务层，并具备 `taskType`；统一顾问若走专用 OpenClaw route，也必须有清晰任务类型和结构化返回约定。

3. 长期资产可确认。
世界书和记忆都不能出现静默写入。

4. 用户可感知。
每个阶段都必须能说明“用户在界面上会看到什么变化”。

## 4. 风险与规避

| 风险 | 说明 | 规避方式 |
|------|------|----------|
| 继续按页面堆功能 | 会重新强化旧叙事 | 先看工具链阶段，再决定落点 |
| 记忆与世界书混淆 | 长期层容易污染 | 先判断内容类型，再选去向 |
| 直接为了补全换富文本框架 | 迁移成本高且未必命中真正问题 | 先收口任务语义和顾问能力，再决定是否上 CodeMirror 6 |
| 分镜先做 UI 后补对象 | 容易形成第二套私有结构 | 先 schema/service，再做页面 |
| 一次改太大 | 容易回归 | 每次只推进一个边界 |

## 5. 最终验收口径

当整套规划完成时，应达到以下状态：

- 体验、素材、卡片画布、写作、世界书、记忆、分镜构成连续工作流。
- 诗歌独立页面已移除；原散文页已成为通用卡片关系画布。
- 长期资产写入可确认、可解释。
- 结构化分镜可版本化、可校验、可导出。
- 页面不再各自维护一套私有生成和输出逻辑。
