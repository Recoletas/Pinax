# 07 - 实施顺序与验收

> 这是落地顺序和检查表，不是新需求。

## 1. 推荐顺序

### 第一批

1. 会话 reset 语义拆分。
2. 会话保存补齐。
3. 世界书切换保护。
4. 去掉调试日志。
5. 补会话测试。

### 第二批

1. 抽 `worldbookContextBuilder`。
2. 做注入预览。
3. 梳理世界书条目层级。
4. 补世界书上下文测试。

### 第三批

1. 体验素材收件箱。
2. 整理本次体验。
3. 写作页消费素材。

### 第四批

1. 统一生成服务。
2. prompt 集中管理。

### 第五批

1. 记忆候选确认流。
2. 作用域隔离。
3. mem0 适配层。

### 第六批

1. 统一 storyboard schema。
2. 版本化。
3. 导出校验。

## 2. 风险

| 风险 | 说明 | 规避 |
|------|------|------|
| 重构过大 | 一次改太多容易回归 | 每批只碰一个边界 |
| 记忆污染 | 自动写入会带来长期偏差 | 所有写入先候选 |
| 世界书注入过多 | token 浪费 | budget report |
| 写作页膨胀 | 页面逻辑越来越重 | 先抽 service |

## 3. 统一验收标准

### 稳定性

- 刷新页面能恢复当前会话。
- 新建会话不会清全局资料。
- 世界书切换不会静默覆盖剧情。

### 写作闭环

- 体验片段能进入素材收件箱。
- 素材能进入章节。
- 写作页能读取体验素材做续写。

### 世界书

- 命中条目可见。
- 常驻条目优先级清晰。
- 截断有预算报告。

### 记忆

- 候选要确认。
- 记忆有 scope。
- 不同作品隔离。

### 编导

- 统一 schema。
- 版本可回退。
- 导出有校验。
