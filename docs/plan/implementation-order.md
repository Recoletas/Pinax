# 07 - 集成实施顺序与验收

> 本文件不是旧式批次流水账，而是把“已完成部分”和“新规划剩余部分”整合后的总阶段表。

## 1. 总体阶段图

| 阶段 | 目标 | 状态 | 说明 |
|------|------|------|------|
| 阶段 A | 会话与运行态稳定 | 完成 | 体验状态边界已建立 |
| 阶段 B | 世界书上下文稳定 | 完成 | 设定注入与预览边界已建立 |
| 阶段 C | 素材中台闭环 | 完成 | 体验到写作已打通 |
| 阶段 D | 统一生成任务层 | 完成 | 业务生成调用已收口 |
| 阶段 E | 规划语言与产品叙事冻结 | 进行中 | 文档与心智从功能页切到工具链 |
| 阶段 F | 编辑器与统一智能顾问收口 | 当前优先级 | 写作页自动补全降级，统一到 OpenClaw 智能顾问 |
| 阶段 G | 记忆系统收尾 | 下一优先级 | 做整理、冲突和同步表达 |
| 阶段 H | 统一分镜输出层 | 下一主线 | 建 schema、版本和导出校验 |
| 阶段 I | 入口与旧结构迁移 | 规划中 | 诗歌/散文降为模式或入口 |

## 2. 当前推荐顺序

### 2.1 现在先完成阶段 E

目标：先把规划语言和剩余工作收束，不再用旧叙事带着实现走偏。

本阶段产出：

- 总计划重写。
- 当前执行计划重写。
- 专题页重写。
- 明确“诗歌/散文不是产品中心，而是工具链入口”。

退出条件：

- 文档体系不再把诗歌和散文写成两条并列产品主线。
- 剩余任务明确收束为编辑器与统一顾问、记忆层、分镜层和信息架构收口四条主线。

### 2.2 接着做阶段 F

目标：先解决写作主链路当前最重的体验问题，把高频自动补全改成更稳的统一智能顾问。

本阶段只做：

1. 决定编辑器路线，不上 ProseMirror。
2. 第一阶段保留 textarea，不做大编辑器迁移。
3. 删掉顾问 AI/LLM 模式，只保留 OpenClaw 统一顾问。
4. 把顾问能力改成自动收线、纠错、局部修复、章节体检。
5. 把自动补全降级为弱能力，默认不再高频自动触发。
6. OpenClaw 不可用时给明确错误和降级说明，不再用用户可见的后端切换兜底。

退出条件：

- 写作主链路不再以 ghost 续写作为核心智能交互。
- 顾问只有一个后端与一条调用链。
- 用户能对选区、段落、章节执行稳定的修正与收线动作。

### 2.3 然后做阶段 G

目标：把记忆系统从“可用”收口成“稳定可控”。

本阶段只做：

1. 候选批量整理。
2. 去重和冲突处理。
3. 同步状态和失败重试表达。

退出条件：

- 候选数量上来后仍可高效处理。
- 本地和远端记忆不会无提示重复或冲突。
- 用户能看见同步结果。

### 2.4 然后做阶段 H

目标：把分镜从页面功能升级成统一输出层。

本阶段顺序：

1. 先建统一 storyboard schema。
2. 再建版本对象和校验器。
3. 再按散文卡片、诗歌分镜树、体验素材、章节片段的顺序接入来源。

退出条件：

- 任一来源都能落成统一 storyboard 对象。
- 历史版本可恢复。
- 导出前有统一校验。

### 2.5 最后做阶段 I

目标：把界面叙事和旧结构收口成真正的工作台模型。

本阶段包含：

- 导航命名调整。
- 入口重排。
- 旧页面语义降级为模式、模板或适配入口。
- 文案与 README 对外叙事统一。

退出条件：

- 新用户能通过导航理解连续工作流。
- 诗歌/散文不再占据主产品心智。

## 3. 每阶段共同门槛

无论推进哪一阶段，都必须同时满足下面四个门槛：

1. 数据归属清楚。
任何新增字段和对象都必须能回答“它属于 session、素材、世界书、记忆还是 storyboard”。

2. 生成链路可追踪。
凡是主模型调用都必须进入统一任务层，并具备 `taskType`；统一顾问若走专用 OpenClaw route，也必须有清晰任务类型和结构化返回约定。

3. 长期资产可确认。
世界书和记忆都不能出现静默写入。

4. 用户可感知。
每个阶段都必须能说明“用户在界面上会看到什么变化”。

## 4. 风险与规避

| 风险 | 说明 | 规避方式 |
|------|------|----------|
| 继续按页面堆功能 | 会重新强化旧叙事 | 先看工具链阶段，再决定落点 |
| 记忆与世界书混淆 | 长期层容易污染 | 先判断内容类型，再选去向 |
| 直接为了补全换富文本框架 | 迁移成本高且未必命中真正问题 | 先收口任务语义和顾问能力，再决定是否上 CodeMirror 6 |
| 分镜先做 UI 后补对象 | 容易形成第二套私有结构 | 先 schema/service，再做页面 |
| 一次改太大 | 容易回归 | 每次只推进一个边界 |

## 5. 最终验收口径

当整套规划完成时，应达到以下状态：

- 体验、素材、写作、设定、记忆、分镜构成连续工作流。
- 诗歌和散文是强力入口，但不再是产品中心。
- 长期资产写入可确认、可解释。
- 结构化分镜可版本化、可校验、可导出。
- 页面不再各自维护一套私有生成和输出逻辑。
