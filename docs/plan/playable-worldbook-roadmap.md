# 可玩的世界书路线图

> 当前产品主线：Pinax 是“可玩的世界书”。用户先进入一个有地图、势力、历史和记忆的世界，由 AI GM 推动冒险；玩出一条剧情线后，再改写成小说、剧本或分镜。

## 1. 定位

Pinax 不应继续被表达为“文字游戏 + 写作工具 + 世界书 + 地图 + 画布”的功能集合。主叙事应收口为一条漏斗：

```text
选择世界
  -> 进入冒险
  -> 形成剧情日志
  -> 写成我的版本
  -> 整理成分镜或改编出口
```

这条路径同时解释三个差异点：

- 相比普通 AI 聊天：Pinax 有世界书、地图、势力和长期上下文。
- 相比 AI Dungeon 类文字游戏：Pinax 的冒险结果能沉淀为可编辑作品。
- 相比纯写作工具：Pinax 不是从空白页开始，而是从“玩一个世界”开始。

短期策略采用“沉浸世界观 + AI GM 控制”。社区 fork 和视频生成都后置，作为漏斗跑通后的加速器和出口能力。

## 2. 非目标

当前阶段不做以下事项：

- 不把短剧/生视频作为主功能优先级；它只作为分镜完成后的改编出口。
- 不先做社区、公开 fork、排行榜或内容市场。
- 不重写世界书 schema、地图引擎或统一生成任务层。
- 不增加新的独立工作区；优先复用体验页、世界书、素材、卡片画布和写作页。
- 不要求第一版具备完整 RPG 系统、战斗系统或复杂数值模拟。

## 3. 阶段计划

### Phase 1 — 入口叙事与首屏路径

目标：用户进入体验页时知道自己不是在“空白聊天”，而是在进入一个世界。

交付：

- 体验页首屏文案改为“进入一个可玩的世界书”。
- 首页或体验页展示三步路径：选择世界、开始冒险、写成作品。
- 当前空白输入态增加“选择一个世界开始”入口，而不是只提示用户输入。
- 世界书、地图、冒险和写作出口用统一术语串联，避免像分散功能。

页面落点：

- `src/components/GamePanel.vue`
- `src/pages/WorldBookQuickImport.vue`
- `src/pages/WorldBookEditor.vue`
- `src/pages/ProseEssay.vue` 仅保留出口提示，不提前主推生视频。

验收：

- 新用户能在 3 步内从体验页进入一个已有世界并开始第一轮冒险。
- 没有世界书时，空态明确引导导入或使用种子世界。
- 生视频入口不抢占冒险/写作主路径。

### Phase 2 — 种子世界

目标：解决冷启动，不要求用户先写设定。

交付 3 个可直接体验的种子世界：

- 边境王国：边境、王权、行会、战争与远征。
- 近未来殖民地：资源、派系、灾害、科技伦理。
- 都市异闻：城市、组织、禁忌、调查与角色关系。

每个种子世界至少包含：

- 世界概述。
- 核心冲突。
- 3-5 个势力。
- 5-8 个重要地点。
- 6-10 个可触发事件。
- 1 个开场困境。
- 3 个后续创作出口提示：小说章节、分镜、扩展世界。

数据落点：

- 优先接入快速导入预设，保持世界书导入三路径可用。
- 不新增不可迁移的临时 schema。
- 地图只作为增强，不作为种子世界启动的硬依赖。

验收：

- 每个种子世界导入后可直接进入体验页冒险。
- 体验页至少 1 轮生成能引用该世界的角色、地点或规则。
- 三条导入路径仍可用：预设导入、小说文本导入、说明驱动 AI 生成。

### Phase 3 — AI GM 与轻状态

目标：让“世界会记住玩家做了什么”成为核心体验。

轻状态字段：

- 当前地点。
- 当前目标。
- 已遇角色。
- 阵营关系。
- 危险等级。
- 关键选择。
- 本段剧情摘要。

交付：

- AI GM 回复前注入当前世界书摘要与轻状态。
- 每轮结束后更新剧情日志和轻状态。
- 机制触发不追求复杂数值，先稳定表达“选择产生后果”。
- 体验页展示当前世界、当前地点、当前目标和最近关键选择。

边界：

- 不引入第二套长期记忆源。
- 不绕开现有生成任务层。
- 状态存储优先使用现有体验页/会话状态，必要时新增轻量结构但不侵入世界书 schema。

验收：

- 玩 10 轮后，系统能生成“你在这个世界经历了什么”的摘要。
- GM 回复稳定引用世界书条目，而不是泛泛续写。
- 切换世界后，不串用上一世界的地点、势力或剧情状态。

### Phase 4 — 玩到写的触发点

目标：补上当前漏斗最大断点。用户玩完一段后，必须自然看到“把它写成我的版本”。

触发按钮：

- 写成我的版本：把本段冒险改写成小说章节。
- 生成另一种结局：把失败或分歧改写成新剧情线。
- 整理成分镜：把关键剧情转成卡片画布时间轴草稿。
- 扩展这个世界：把新角色、新地点、新事件写回世界书草稿。

触发时机：

- 每 8-12 轮出现一次“本段冒险总结”。
- 失败、危险升级或关键选择后出现“改写结局/生成分支”。
- 会话结束或用户暂停时出现“保存为作品草稿”。

服务落点：

- 冒险日志 -> 写作页草稿。
- 冒险日志 -> 素材节点。
- 冒险日志 -> 卡片画布时间轴草稿。
- 冒险中新出现的设定 -> 世界书草稿预览。

验收：

- 用户完成一段冒险后，至少一个主按钮会引导到写作或分镜，而不是只剩“继续聊”。
- 生成的章节或分镜能保留地点、角色、关键选择和结局。
- 用户能拒绝草稿，不会被强制写回世界书或素材库。

### Phase 5 — 改编出口

目标：证明从可玩的世界书可以走到分镜和短片，但不把短剧生产作为当前核心。

交付顺序：

1. 冒险日志整理成卡片画布时间轴。
2. 分镜版本校验与导出继续稳定。
3. 生视频功能作为分镜完成后的实验出口接入。

生视频定位：

- 名称建议为“生成视频草稿”或“短片实验包”。
- 只在 directing 模式、有分镜版本时启用。
- 不常驻抢占体验页和写作页主路径。
- MVP 只要求镜头级生成、顺序预览、manifest/ZIP 导出，不要求真实 MP4 拼接。

验收：

- 至少一个 demo case 能从种子世界冒险走到小说片段、分镜和视频草稿出口。
- 用户没有走完冒险/写作链路前，不会被生视频入口分散注意力。

## 4. 第一轮实施范围

第一轮只做 Phase 1 的可见改动和 Phase 2 的种子世界基础，不碰后置出口。

第一轮交付：

- 体验页空态和首屏文案改成“进入可玩的世界书”。
- 没有世界书时引导使用快速导入预设。
- 快速导入预设补齐 3 个种子世界入口。
- 冒险页增加“本次冒险可写成作品”的轻提示，但不实现完整日志转写。
- 卡片画布/生视频只保留为远景出口，不作为主 CTA。

第一轮不做：

- 不做社区 fork。
- 不做视频生成。
- 不做完整 GM 状态机。
- 不做短剧剧本层。

## 5. 验证策略

自动化：

- 世界书快速导入预设测试：3 个种子世界可生成预览并导入。
- 体验页空态/入口测试：没有世界书时显示导入引导。
- 路由/页面 smoke：体验页、世界书快速导入、世界书高级编辑仍可挂载。

手动：

- 从空白状态进入体验页，确认能被引导到种子世界。
- 导入一个种子世界，回到体验页完成至少 1 轮冒险。
- 检查体验页文案是否强调“进入世界 -> 冒险 -> 写成作品”。
- 检查卡片画布和生视频不成为首屏主 CTA。

## 6. 决策记录

- 入口差异点采用“沉浸世界观 + AI GM 控制”，不单独押注社区 fork。
- 短剧/生视频是改编出口，不是当前 product goal。
- 种子世界优先于复杂配置，让用户先玩起来。
- 玩到写的触发点比出口能力更优先。
