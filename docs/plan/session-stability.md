# 01 - 会话与运行态边界

状态：基础已完成

## 1. 在工具链中的位置

会话系统不是页面附属功能，而是体验阶段的状态底座。

它负责回答三个问题：

1. 当前这次体验到底保存了什么。
2. 哪些内容属于本次会话，哪些属于全局写作资料。
3. 切换、刷新、重写和世界书切换时，哪些内容应该保留，哪些应该重置。

## 2. 已完成的边界定义

| 资产 | 所属边界 | 当前规则 |
|------|----------|----------|
| `runtimeState` | 会话内 | 跟随 session 保存、恢复和切换 |
| 消息 / 对话历史 | 会话内 | 跟随 session 持久化 |
| 当前世界书绑定 | 会话内 | 保存到 session，但切换需显式确认 |
| 写作角色 / 时间 / 地图 / 活动 | 会话内运行态 | 跟随 session 持久化和恢复 |
| 全局写作资料 | 全局层 | 不因新建或切换会话被清空 |

已经落地的控制点：

- 会话运行态与全局写作资料已拆分。
- `createSession()` 支持明确配置，默认新建空运行态。
- 保存/恢复覆盖消息、历史、世界书、角色、时间、地图、活动等关键运行态。
- 新建、切换、删除、重写后续等关键路径都有保存点。
- 世界书切换有保护，不再静默覆盖当前剧情。

## 3. 这个模块现在不再是什么

- 不再是“顺手存在 store 里的页面状态集合”。
- 不再负责存放全局写作资料。
- 不再允许再长出第二套体验主状态源。

## 4. 后续只允许做的事

这个专题已经不再是主动扩展项，后续只允许三类改动：

1. 修回归 bug。
例如新建会话后初始化异常、恢复内容缺字段、切换世界书提示缺失。

2. 做兼容性补丁。
例如 schemaVersion 升级、旧 session 数据迁移、缺字段兜底。

3. 跟随上游对象增加小范围字段。
例如 runtimeState 新增必须持久化的字段，但前提是不破坏边界。

## 5. 不应再做的事

- 不继续扩写第二套体验 store。
- 不把候选设定、长期记忆、素材列表直接塞回会话对象。
- 不用会话层去代替素材中台或记忆系统。

## 6. 验收与回归清单

| 验收项 | 当前状态 |
|--------|----------|
| 新建会话不清全局资料 | 完成 |
| 刷新后恢复当前会话 | 完成 |
| 删除消息不会“回魂” | 完成 |
| 重写后续可持久化 | 完成 |
| 世界书切换不静默覆盖剧情 | 完成 |

回归时优先检查：

1. prompt 传参是否引用了旧运行态。
2. session 快照是否遗漏新字段。
3. 页面是否绕开 store 直接改存储。

## 7. 与后续主线的关系

- 记忆系统后续只能读取会话内容生成候选，不能反过来让会话直接承担长期记忆职责。
- 分镜系统后续可以读取会话片段或体验素材，但不应把 storyboard 版本直接存进 session 运行态。
