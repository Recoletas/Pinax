# 当前执行计划

> 本文件是继续推进时的默认入口。先看这里，再决定是否打开专题页。当前阶段不再按“诗歌页还差什么、散文页还差什么”来排期，而是按工具链缺口来排期。

## 1. 当前结论

粗略进度：90%

已完成的不是零散功能，而是四块基础设施：

1. 会话和运行态边界已经稳定。
2. 世界书上下文和素材中台已经形成闭环。
3. 生成调用已经统一到任务层。
4. 诗歌、散文、体验、顾问都已经纳入统一生成体系。

因此，真正剩下的只剩三条线。当前按整体优化优先，先推进分镜输出层，不继续把时间压在记忆细节上：

1. 统一分镜输出层。
2. 产品叙事和信息架构从“功能页”切到“工具链”。
3. 记忆系统只做必要维护和轻量补丁。

## 3. 当前执行顺序

### 3.1 先完成：规划语言和边界冻结

这一步已经通过本轮文档重写启动，目标是先把“项目到底是什么”说清楚，避免后续继续按旧页面心智新增实现。

当前要求：

- 文档统一使用“工具链、素材、设定、记忆、分镜”的主语言。
- 诗歌、散文在规划层不再被写成两个产品中心。
- 所有剩余任务必须能挂到工具链阶段，而不是只挂到某个页面。

### 3.2 已完成：编辑器与统一智能顾问

这条线已经落地，当前写作页仍保留 textarea，但智能交互已统一到 OpenClaw 顾问。

已完成的结果：

1. 顾问收口为单后端，不再暴露 AI / OpenClaw 切换。
2. 顾问任务改成结构化动作，支持选区修正、段落修正、收线、章节体检和轻续写。
3. 可应用结果已接入写作页，替换型建议会走校验和应用流程。
4. `useCopilot` 已改成弱补全模式，默认不自动触发。

当前细节见专题：[editor-advisor-unification.md](./editor-advisor-unification.md)

### 3.3 已完成到可暂缓：记忆系统

记忆系统已经完成到可用状态，后续只做必要维护，不再作为当前推进重心。

本线只做四类事：

1. 候选批量整理。
2. 本地与远端记忆去重。
3. 冲突表达和确认策略。
4. 同步状态和失败重试的可见性。

不做的事：

- 不把记忆继续做成自动写入系统。
- 不把记忆和世界书混成同一种资产。
- 不在页面里零散加同步提示，先确定统一状态表达方式。

当前同步状态的最小可见性已经补上，待确认项也已支持批量选择、批量确认、批量拒绝、批量改 scope、批量归档、已归档视图、基础冲突提示、替换旧记忆、合并旧记忆、本地相似提示、远端精确去重和同 scope 的规则型重复提示。

后续暂缓细项：

- 排序筛选细化。
- 更强语义去重。

### 3.4 当前主线 C：统一分镜输出层

这是下一块真正新的系统性工作，目标是把诗歌、散文、体验片段和章节片段统一收束到同一套 storyboard 结构。

本线分三步：

1. 先定义 schema、版本对象和校验规则。（已启动）
2. 再做适配层，散文、诗歌、体验素材和章节草稿的入口已经接完，后面主要收导出校验和规则。
3. 最后再做导出和界面迁移，不反过来。

当前已接入散文、诗歌、体验素材和章节草稿的直接导出，主入口导出也已经统一到校验保存入口，后续重点转到更细的规则收口，细节多的部分可以先写进计划，暂不展开实现。

不做的事：

- 不先做 UI 大改后补底层对象。
- 不继续扩诗歌和散文各自的独立分镜格式。
- 不把导出做成页面私有功能。

### 3.5 后续主线 D：产品信息架构收口

当前已经开始做导航和页面入口收口，后续继续把素材、设定、分镜这些主线入口补齐并统一命名。

方向：

- 一级导航优先体现体验、写作、素材、设定、分镜、顾问。
- 诗歌和散文作为写作/分镜下的模式、模板或入口存在。
- 用户最终看到的是连续工作流，而不是多个互不相干的小工具页。
