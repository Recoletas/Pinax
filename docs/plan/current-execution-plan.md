# 当前执行计划（UI 可用性优先）

> 本文件是当前唯一执行面板。按优先级完成 UI 可用性收口后，再继续新增能力。兼容性不是第一目标，只保留为“不能阻塞用户完成任务”的回归底线。

## 1. 当前执行结论

当前阶段完成度不按功能数衡量，按“用户能不能顺利完成主任务”衡量。

当前重点：

1. 先收口核心任务路径，减少重复入口。
2. 再清理跨页面 UI 冲突和移动端阻塞。
3. 再统一交互结构。
4. 最后做视觉细化和文案统一。

决策顺序：

```text
任务可完成 > 入口清晰 > 页面一致 > 移动端不阻塞 > 旧入口兼容 > 视觉细节
```

## 3. 当前迭代任务（可直接执行）

### 3.1 P0-A：核心路径可完成

目标：先保证用户能不迷路地完成主任务，而不是保留所有旧入口。

重点路径：

- 体验：开始/继续体验 -> 速记或整理素材 -> 进入写作、素材或设定页。
- 写作：选择章节 -> 查看素材 -> 插入正文/入纲要/入世界书 -> 保存。
- 设定：普通导入 -> 预览冲突 -> 确认写入；高级设置作为次级入口。
- 分镜：从写作或素材触发草稿 -> 校验 -> 保存版本或导出。

验收：

1. 每条路径最多保留 1 个主入口 + 1 个就近入口。
2. 用户不需要理解“笔记/素材/草稿/导出”的旧差异才能完成任务。
3. 低频入口可以合并、移动或暂时降级，不以旧位置兼容为硬要求。

首轮进展：

- 体验页顶部已收口为“整理素材 / 分镜 / 会话 / 主题 / 设置”，AI 开关从界面移除。
- 体验页速记保存已收口为单一“保存素材”，不再并列两条素材保存路径。
- 写作页顶部移除重复的新建书籍，保留“收件箱”和“章节分镜”两个任务入口。
- 素材页已切到统一 `narrativeAssets` 资产池，与写作页收件箱共用同一批素材。

### 3.2 P0-B：固定定位冲突收口

目标：解决多固定入口互相遮挡、滚动冲突。

重点页面：

- `src/pages/Experience.vue`
- `src/pages/Writing.vue`

验收：

1. 同页同时开启多个浮层时，层级和焦点可预测。
2. 浮层打开后背景滚动策略一致，不出现“滚不动”或“穿透滚动”。
3. 360px 宽度下关键按钮可见且可点击。

### 3.3 P0-C：视口与高度策略统一

目标：消除 `100vh` 在移动端软键盘场景的布局抖动。

重点页面：

- `src/pages/Writing.vue`
- `src/layouts/AppShell.vue`
- `src/pages/Experience.vue`

验收：

1. 输入法弹出时主编辑区可见，不被固定头部和底部挤压到不可编辑。
2. 横竖屏切换后不出现内容截断。

### 3.4 P1-A：高频操作路径减负

目标：减少“同一功能多入口”导致的认知负担。

范围：

- 体验页顶部按钮、悬浮入口、速记入口。
- 写作页顶部工具、素材入口、分镜入口。

验收：

1. 每个高频任务最多保留 1 个主入口 + 1 个就近入口。
2. 同类动作文案和位置在不同页面保持一致。

## 4. 当前阶段禁止事项

- 不新增新的固定悬浮入口。
- 不在 UI 未收口前继续扩展复杂功能面板。
- 不通过增加文案解释来替代交互简化。
- 不为了兼容旧按钮位置而保留重复入口。
- 不把“所有功能都露出”当成可用性。

## 5. 回归检查清单

每轮 UI 改动后，至少完成以下检查：

1. 视口：360 / 768 / 1280 三档。
2. 主题：亮色与暗色。
3. 交互：键盘输入、面板开关、滚动与关闭。
4. 核心路径：体验到素材、素材到写作、世界书导入、分镜保存。
5. 构建：`npm run verify`。
