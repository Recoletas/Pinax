# 当前执行计划（UI 可用性优先）

> 本文件是当前唯一执行面板。按优先级完成 UI 可用性收口后，再继续新增能力。兼容性不是第一目标，只保留为“不能阻塞用户完成任务”的回归底线。

## 1. 当前执行结论

当前阶段完成度不按功能数衡量，按“用户能不能顺利完成主任务”衡量。

当前重点：

1. 先把素材页收口成轻量分组的资产中转层。
2. 再把原散文卡片页收口成以关系和时间轴为中心的卡片画布。
3. 将分镜变成卡片画布中的制作视图和输出链路。
4. 把右上连线图例、节点详情和时间轴继续压到 VS Code Git 式的信息密度。
5. 移除不再支撑主路径的诗歌独立工作区，并收尾跨页面 UI 冲突、移动端阻塞和视觉细节。

决策顺序：

```text
任务可完成 > 入口清晰 > 页面一致 > 移动端不阻塞 > 旧入口兼容 > 视觉细节
```

### 1.1 当前产品决策

新的核心链路：

```text
体验所得内容
  -> 素材中转：整理、编辑、分类、生成或保存图片
  -> 卡片画布：展示素材引用、空间关系、分组和镜头顺序
  -> 分镜版本：校验、保存、导出
```

职责边界：

| 模块 | 保留职责 | 不再承担 |
| --- | --- | --- |
| 素材 | 原始文本/图片内容、来源、状态、基础分类、跳转卡片 | 重型关系图、过密标签说明 |
| 卡片画布（原散文卡片页） | 素材节点、预览图、关系线、分组、顺序、镜头参数 | 长文编辑器、重复素材库、以散文为中心的叙事 |
| 分镜 | 基于画布关系形成镜头序列、校验和导出 | 独立复制另一套内容 |
| 诗歌 | 不作为独立工作区继续建设 | 页面级意象树和私有分镜出口 |

诗歌页判断：

- 它唯一可能保留的价值，是按意象和联想扩散产生素材关系。
- 这个价值不要求独立页面，可以下沉为卡片画布中的一种“意象扩散”生成动作。
- 当前主路径中，诗歌页会迫使用户理解一套额外的意象树、版本和分镜切换，成本高于价值。
- 决策：停止诗歌页继续扩展；先迁走可复用生成动作，随后移除入口、路由、页面和专属提示词。旧数据迁移/导出只在实现成本低时保留，不作为阻塞条件。

## 3. 当前迭代任务（可直接执行）

### 3.1 P0-A：核心路径可完成

目标：先保证用户能不迷路地完成主任务，而不是保留所有旧入口。

重点路径：

- 体验：开始/继续体验 -> 速记或整理素材 -> 进入写作、素材或世界书页。
- 写作：选择章节 -> 查看素材 -> 插入正文/加入纲要/入世界书 -> 保存。
- 世界书：普通导入 -> 预览冲突 -> 确认写入；高级编辑作为次级入口。
- 分镜：从素材跳转到卡片画布 -> 建立关系/镜头顺序 -> 校验 -> 保存版本，用户确认后导出。

验收：

1. 每条路径最多保留 1 个主入口 + 1 个就近入口。

### 3.6 P0-E：关系画布重构（原散文卡片页）

目标：卡片页成为素材的立体关系视图，关系可读性优先于单卡内容展示。

信息结构：

- 主画布：卡片节点、连接线、关系类型、分组、镜头顺序。
- 节点：小预览图、短标题、来源/镜头标记；正文最多显示一行弱预览。
- 选中详情：完整素材入口、镜头参数、加入/移出分镜序列和删除；关系编辑直接在画布完成。
- 时间轴/分镜层：镜头顺序和转场是主信息，正文只作为辅助预览。

数据边界：

- 卡片新增或收口为 `assetId` 引用，内容从 `narrativeAssets` 读取。
- 画布持久化位置、关系、分组、选中状态和镜头参数。
- 图片保存在素材资产上；卡片节点只消费缩略预览，不再维护独立图片库。
- 现有散文自由生成可转换成“生成素材并加入画布”，不再生成页面私有长文卡。

简化项：

- 暂不做多图管理、复杂搜索、关系自动推断和大规模图谱布局。
- 第一轮只完成素材跳转、引用节点、清晰连接线、紧凑节点和镜头顺序。

验收：

1. 首屏视觉中心是关系线与节点结构，而不是卡片正文。
2. 节点即使没有长文也可通过标题、图片和关系被识别。
3. 用户可以从素材进入画布、连线、排序并生成/保存分镜版本。

3. 素材内容、卡片关系和分镜版本状态分离，查看分镜不会复制或污染素材正文。

### 3.9 P1-D：模式切换门控

目标：避免“有字但内容还没准备好，却被迫进入分镜态”的误导。

规则：

- 当画布已有卡片关系但没有分镜版本时，不允许把预览误称为完成的“分镜版本”。
- 这时应优先显示“生成分镜版本”按钮，或要求用户先确认当前内容可作为分镜输入。
- 只有在存在可用分镜版本后，分镜态才允许进入常驻展示。
- 如果用户仍想强切，必须显式确认“将当前内容作为分镜草稿输入”，不能静默切换。

验收：

1. 用户不会在没有结果卡片的情况下误以为已经完成分镜切换。
2. 需要输入确认的场景必须有明确提示，不依赖文案解释。
3. 分镜态入口状态应能反映当前是否已有可用版本。

### 3.10 P1-E：分镜画布可视化

目标：分镜侧把画布作为主体验，而不是只在侧栏堆文案和按钮。

范围：

- 卡片画布需要突出关系线、节点位置、分组、镜头顺序、版本态和校验态。

验收：

1. 分镜态的首屏主视觉是画布或画布式版本视图。
2. 版本卡片、校验和下载是辅助，不是唯一主操作。
3. 用户可以通过画布上的节点/卡片直接理解当前分镜的结构关系。

### 3.11 P1-F：画布逻辑复用

目标：优先复用已有卡片的拖拽、连线和时间轴能力，将其收成单一关系画布，不再维护诗歌侧第二套画布。

可复用能力：

- 节点拖拽与手动排布。
- 连线创建、删除和边类型切换。
- 节点分组 / 牌堆。
- 选中态、hover 态和局部编辑。
- 画布缩放、平移和视口定位。

追加能力：

- 分镜版本卡片与版本切换。
- 从当前画布直接生成分镜版本。
- 分镜态只读时保留浏览与下载，不再允许误导性切换。
- 卡片关系图保持主视觉，分镜侧叠加镜头顺序和转场信息。

验收：

1. 分镜侧的主视觉仍来自原有画布，不出现第二套完全不同的编辑器。
2. 新增功能只包在现有节点/卡片/连线体系上。
3. 用户从关系视图切到分镜视图时，节点位置和连接关系保持连续。

### 3.12 P1-G：预览图与镜头信息

目标：降低纯文本分镜的占比，但不让图片管理压过关系结构。

当前决策：

- 卡片节点内只显示小型预览图，不再显示大参考图位；有缩略图时正文预览隐藏，画布顶部的横向预览条已经删掉。
- 卡片节点的图片引用现在统一跟随素材或附件元数据走，导出阶段只保留轻量引用信息。
- 图片生成、保存和完整预览都在素材页处理。
- 卡片画布只显示引用到的图片缩略图，以及景别、运镜、时长、转场等必要标记。
- 无图卡片维持单行弱预览，不另起一层空状态控件。
- 右上关系线图例的默认收起态以标题和两个图标按钮呈现，避免连线和删线按钮在小面板里互相挤压。
- 时间轴条目开始承接画布关系线信息，已有上一镜关系时用短文本提示关系类型，导出时间轴元数据保留关系字段。
- 导出区域不再只是动作列表，而是带版本状态的轻量反馈区；时间轴头部直接给出生成/更新/下载主动作，平时不必先打开导出菜单。
- 剪辑包导出现在直接生成 ZIP，包含 shots、manifest、可拆分文件清单、Premiere CSV、剪映草稿和 FCP XML，素材、关系和参考图元数据不会在格式转换时丢失。
- 诗歌独立入口已经退场，后续只清底层兼容代码和必要迁移动作。

下一步：

- 下一步优先从工作流层面收口：把真实外部剪辑工具对接、素材生成入口和分镜版本消费排好顺序，不再继续围绕导出按钮做细节微调。
- 视情况再收紧节点详情里的按钮数量，优先保留最常用动作。

暂缓：

- 多图切换和复杂图片管理。
- 自动生成关系或自动布局。

### 3.13 P1-H：素材生图闭环

目标：素材页不只是文本收件箱，而是图片素材、正文插图和分镜参考图的统一入口。

已完成：

- 增加素材页生图入口。
- 生成图可保存为 `reference-image` 素材。
- 生成图可直接插入正文，并以 Markdown 图片形式在预览中显示。
- 选中图片素材可送入分镜。
- 分镜页可直接读取参考图素材和分镜种子。

下一步：

- 精简左侧列表后，在详情区保留图片预览和生图入口。
- 增加加入/定位卡片画布的动作，使图片素材也能成为关系节点。
- 再细一点的分类筛选、批量操作和排序暂缓。

## 4. 当前阶段禁止事项

- 不新增新的固定悬浮入口。
- 不在 UI 未收口前继续扩展复杂功能面板。
- 不通过增加文案解释来替代交互简化。
- 不为了兼容旧按钮位置而保留重复入口。
- 不把“所有功能都露出”当成可用性。

1. 视口：360 / 768 / 1280 三档。
2. 主题：亮色与暗色。
3. 交互：键盘输入、面板开关、滚动与关闭。
