# 02 - 世界书上下文与设定回写

状态：基础已完成

## 1. 在工具链中的位置

世界书层负责“设定输入”和“设定沉淀”，它既不是普通文本缓存，也不是长期记忆的替代品。

工具链中的位置如下：

```text
世界书 / 设定
	-> 参与体验与写作生成上下文
	-> 体验/写作产出的新事实先变成草稿或候选
	-> 作者确认后再进入正式世界书
```

## 2. 已完成的基础设施

- 已抽出统一的世界书上下文构建器。
- 上下文构建支持 runtimeState、关键词扫描、常驻条目、高优先级条目和 token budget。
- 体验生成、Copilot 等链路已接入统一世界书上下文。
- 提示词详情和世界书预览链路已接入，用户可以看到命中条目和上下文内容。
- 世界书草稿不再自动写回正式世界书，而是经由素材收件箱或显式确认入库。

## 3. 已冻结的核心规则

| 规则 | 当前口径 |
|------|----------|
| 正式世界书 | 只存放已确认、应长期参与生成的设定 |
| 世界书草稿 | 先作为待确认资产存在，不直接污染正式世界书 |
| 常驻条目 | 优先保证进入上下文，但仍受 budget 控制 |
| 命中条目 | 必须可预览、可解释，而不是黑盒注入 |
| 截断行为 | 必须能报告预算，而不是静默丢弃 |

## 4. 这个模块已经解决的问题

1. 世界书注入顺序稳定。
2. 常驻和高优先级条目能稳定优先进入上下文。
3. 用户可以看到命中内容，不再是纯黑盒关键词注入。
4. 体验和写作产出的新设定不会直接污染正式世界书。

## 5. 后续只做的事

这个专题目前只做补强，不再做大重构。

允许继续做：

1. 优化预算报告展示。
2. 补充更多条目类型的 UI 解释。
3. 提高草稿确认流的可读性。

不建议现在做：

- 重写核心 context builder。
- 回到自动写入世界书的旧路径。
- 让世界书承担记忆系统的职责。

## 6. 和记忆系统的边界

后续必须继续区分世界书与记忆：

| 情况 | 去向 |
|------|------|
| 稳定世界设定、角色设定、地点设定、物品设定 | 世界书草稿 -> 正式世界书 |
| 作者偏好、作品长期约束、阶段性工作上下文 | 记忆候选 -> 记忆系统 |
| 尚未确认、仍需斟酌的内容 | 素材或候选，不直接入世界书 |
