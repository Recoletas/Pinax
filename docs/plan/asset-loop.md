# 03 - 素材中转层与关系画布闭环

状态：基础已完成

## 1. 在工具链中的位置

素材层是整个项目的内容中转层和资产真源。体验、写作、世界书、卡片画布、分镜之间不应该直接互抄内容，而应该优先通过素材流转。

当前目标闭环：

```text
体验消息 / 会话摘要 / 手动记录 / 生图结果
	-> NarrativeAsset
	-> 收件箱整理
	-> 正文 / 纲要 / 世界书草稿 / 续写参考
	-> 卡片画布引用
	-> storyboard version
```

## 2. 已完成的能力

- 已有统一素材服务 `src/services/narrativeAssets.js`。
- 体验页速记、选段和“整理体验”都可以生成素材。
- 写作页素材收件箱已收口为工作面板，而不是散落动作。
- 素材支持范围筛选、类型筛选、单条处理和批量处理。
- 世界书草稿可显式写入当前世界书。
- 素材主链路统一走 `narrativeAssets`；旧笔记服务仅保留兼容，不再作为主路径。
- 素材可作为 Copilot 续写参考。
- 素材可进入当前章节纲要，并参与内联续写上下文。
- 页面可以写入 `storyboard-seed` 或 `reference-image`，素材详情现可创建并定位卡片画布引用节点。

## 2.1 新职责边界

| 层级 | 保存什么 | 主要动作 |
|------|----------|----------|
| 素材 | 完整文本、图片、来源、类型、采用状态 | 整理、分类、状态处理、生图、按选中/批量加入卡片画布 |
| 卡片画布 | `assetId`、位置、关系、分组、镜头参数、顺序 | 可视化关系、编排镜头 |
| 分镜版本 | 由画布提取的镜头快照和校验结果 | 保存、回看、导出 |

规则：

- 卡片节点不应继续拥有与素材并行维护的长正文副本。
- 图片原文件与提示词归素材；画布只显示缩略预览和引用。
- 素材列表不展示完整分类解释；说明放到筛选或详情中。

## 4. 当前素材生命周期

| 状态 | 含义 |
|------|------|
| `inbox` | 新进入收件箱，尚未决定去向 |
| `accepted` | 已被采用并流向正文、纲要、世界书等 |
| `rejected` | 明确不采用 |
| `archived` | 保留但不参与当前工作流 |

## 5. 当前这条线还缺什么

这个专题主体已经完成，但为了与新规划对齐，后续还应补两类内容：

1. 更好的批量整理体验。
主要是列表效率、批量状态切换、批量分配目标去向。

2. 对卡片画布的消费继续收口。
`storyboard-seed` 已可创建引用节点；下一轮重点是从引用关系稳定生成、校验和保存分镜版本。

参考图素材当前先走轻量闭环：

- 生图结果可以保存为 `reference-image` 素材。
- 素材页展示图片预览，但不在这一轮扩展复杂图片管理。
- 素材页的图片可直接插入正文，并在预览里显示。
- 卡片画布从素材读取缩略图并显示在紧凑节点中，不在节点内部管理图片。

## 6. 与旧诗歌 / 散文页面的关系

新规划下，不再把诗歌和散文作为素材层两侧的独立工作区：

- 原散文卡片能力重命名并改造成卡片画布，节点必须逐步绑定素材引用。
- 诗歌页没有独立主路径必要性；仅在成本低且确有帮助时，把“意象扩散”迁成卡片画布的生成动作。
- 诗歌私有意象树与分镜出口迁移后删除，不再继续增强。

这意味着新增关系或分镜能力首先应回答“它是否属于卡片画布”，而不是再开一个题材页面。
