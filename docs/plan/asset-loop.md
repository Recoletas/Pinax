# 03 - 体验到写作素材闭环

> 目标：把体验页生成的内容沉淀成可继续加工的写作资产。

## 1. 当前问题

现在体验页已经有速记、导入对话、跳转写作、跳转笔记等动作，但这些动作还不够结构化：

- 保存内容主要还是“笔记”。
- 体验片段没有明确分类。
- 体验结果和章节正文之间没有稳定桥梁。
- 体验输出没有形成统一素材收件箱。

## 2. 目标

把体验、笔记、写作、世界书草稿连接成一个素材流：

```text
体验消息
  → 选段
  → 素材收件箱
  → 写作章节 / 世界书草稿 / 分镜种子 / 人物事实 / 剧情事件
```

## 3. 素材模型

建议先建立统一素材结构，而不是分散在多个 key 里。

```ts
type NarrativeAsset = {
  id: string
  schemaVersion: number
  projectId: string | null
  source: {
    type: 'experience-session' | 'note' | 'chapter' | 'manual'
    id: string
    messageIds?: string[]
  }
  kind: 'draft-prose' | 'event' | 'character-fact' | 'worldbook-draft' | 'inspiration' | 'storyboard-seed'
  title: string
  content: string
  status: 'inbox' | 'accepted' | 'rejected' | 'archived'
  createdAt: number
  updatedAt: number
}
```

## 4. 具体方案

### 4.1 体验选段扩展

当前的“导入对话段”建议扩展成素材类型选择。

保存时至少支持：

- 正文候选
- 剧情事件
- 人物事实
- 世界书草稿
- 灵感
- 分镜种子

### 4.2 新增素材收件箱

素材先进入 inbox，不直接进入章节正文。

收件箱至少要支持：

- 查看来源
- 修改标题
- 修改类型
- 接受
- 拒绝
- 归档

### 4.3 整理本次体验

新增一个总结动作，AI 输出：

- 剧情摘要
- 角色变化
- 未解决悬念
- 可用正文片段
- 可写入世界书的候选设定

结果先进入素材收件箱，由作者选择后续去向。

### 4.4 写作页消费素材

写作页应该能读取素材收件箱，并支持：

- 插入正文
- 作为续写上下文
- 作为章节大纲素材
- 转成笔记

## 5. 实施步骤

### 第一步

- 建一个统一素材存储层。
- 让体验选段保存到素材对象。

### 第二步

- 做素材收件箱。
- 给素材增加类型和来源信息。

### 第三步

- 新增“整理本次体验”。
- 输出摘要、悬念、正文候选。

### 第四步

- 写作页接入素材。
- 支持插入和续写。

## 6. 验收标准

- 体验片段能按类型沉淀。
- 素材来源可追溯。
- 写作页能直接消费体验素材。
- 世界书草稿不会直接污染正式世界书。
