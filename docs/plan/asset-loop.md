# 03 - 素材中台与体验到写作闭环

状态：基础已完成

## 1. 在工具链中的位置

素材层是整个项目的交换层。体验、写作、设定、记忆、分镜之间不应该直接互抄内容，而应该优先通过素材中台流转。

当前目标闭环：

```text
体验消息 / 会话摘要 / 手动记录
	-> NarrativeAsset
	-> 收件箱整理
	-> 正文 / 纲要 / 世界书草稿 / 素材条目 / 续写参考
	-> 后续扩展到 storyboard seed
```

## 2. 已完成的能力

- 已有统一素材服务 `src/services/narrativeAssets.js`。
- 体验页速记、选段和“整理体验”都可以生成素材。
- 写作页素材收件箱已收口为工作面板，而不是散落动作。
- 素材支持范围筛选、类型筛选、单条处理和批量处理。
- 世界书草稿可显式写入当前世界书。
- 素材主链路统一走 `narrativeAssets`；旧笔记服务仅保留兼容，不再作为主路径。
- 素材可作为 Copilot 续写参考。
- 素材可进入当前章节纲要，并参与内联续写上下文。

## 3. 现在的素材层已经承担的职责

| 来源 | 当前是否已接入 | 当前去向 |
|------|----------------|----------|
| 体验页选段 | 已接入 | 正文候选 / 事件 / 事实 / 世界书草稿 / 灵感 |
| 体验整理总结 | 已接入 | 摘要类资产和候选设定 |
| 手动处理 | 已接入 | 标题、类型、状态整理 |
| 写作页消费 | 已接入 | 正文 / 续写参考 / 章节纲要 / 素材条目 / 世界书 |

## 4. 当前素材生命周期

| 状态 | 含义 |
|------|------|
| `inbox` | 新进入收件箱，尚未决定去向 |
| `accepted` | 已被采用并流向正文、纲要、世界书等 |
| `rejected` | 明确不采用 |
| `archived` | 保留但不参与当前工作流 |

## 5. 当前这条线还缺什么

这个专题主体已经完成，但为了与新规划对齐，后续还应补两类内容：

1. 更好的批量整理体验。
主要是列表效率、批量状态切换、批量分配目标去向。

2. 对分镜系统的直接消费。
后续 storyboard 统一后，`storyboard-seed` 应该成为素材的一类正式出口，而不是再在诗歌或散文页里私有处理。

## 6. 与诗歌 / 散文的关系

新规划下，诗歌和散文不再是素材层的并列对手，而是素材层的上游或下游：

- 上游：它们可以生成可复用的文本片段或分镜种子。
- 下游：它们也可以消费素材作为创作输入。

这意味着后续如果继续增强诗歌或散文能力，优先问题不是“页面多一个按钮”，而是“它是否应该经过素材层沉淀”。
