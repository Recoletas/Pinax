## Summary

V1/V2 修掉了"土"(去切角/去大写/统一边框),但做了过度减法,把档案册性格也砍了,顶部 tab 变成 Material 通用风 + 灰框圆角 = 跟任何 SaaS 一样普通。V3 给档案册找 1 个贯穿全局的签名手法:**印章符号 `◆` + 印泥 `archive-rose` + 罗马序号 + paper-fiber 底 + 撕边虚线分隔**,让"档案册"回归但不拼贴不土。

5 项签名手法改动 + 删 V2 引入的 2px SaaS 滑动线。

## 5 项签名手法改动

### 1. tab active 印章 `◆`

**删 V2 的 SaaS 滑动线**:`.shell-tab-indicator` 2px accent 滑动条(V2 commit `42ed7f3` 加的)+ 相关 CSS + JS `updateTabIndicator()` / `tabbarRef` / `indicatorStyle` / `handleTabbarScroll` / resize + scroll 监听全删,共移除约 25 行 JS。

**新印章语言**:active tab 前缀 `◆` 符号 archive-rose 8-10px,通过 `::before` 伪元素实现:
- 非 active tab:`content: ""; display: inline-block; width: 0; opacity: 0`(占位但不可见)
- active tab:`content: "◆"; opacity: 1; width: auto; margin-right: 4px; font-size: 9px; line-height: 1; color: color-mix(in srgb, var(--accent-rose) 82%, transparent)`
- 背景仍透明(不靠底色高亮)
- label 字重从 500(inactive)→ 600(active),纯字重 bump,不靠 underline/glow/box-shadow
- `aria-hidden="true"` 加在 `.shell-tab__index` 上(印章符号靠 `aria-selected` 表态)

### 2. tab 序号罗马数字 + LXGW

**Script**:在 `AppShell.vue` script 段加常量 `const ROMAN_ACTIVITY_STAMPS = ['Ⅰ', 'Ⅱ', 'Ⅲ', 'Ⅳ', 'Ⅴ']`(硬编码 5 项,activity 数增长再换 toRoman)。

**Template**:`<span class="shell-tab__index" aria-hidden="true">{{ ROMAN_ACTIVITY_STAMPS[index] || '·' }}</span>`(去掉 `String(index + 1).padStart(2, '0')`)。

**CSS**:
- `font-family: var(--font-display, "Iowan Old Style", "Songti SC", "STSong", Georgia, serif)` — LXGW WenKai 书法体
- `font-size: 12px`(mobile 760 降 11px)
- `color: var(--text-muted)` / active → `var(--text-primary)`
- 删 V1 已删的 `transform: skewX(16deg)`(确认无)
- 删 `font-variant-numeric: tabular-nums`(罗马数字不需要等宽)
- mobile 760 不再 `display: none` 隐藏 index(罗马序号在 mast 折叠时仍可读)

### 3. chip 印章墨点 + 直角边框

**`shell-meta-chip` / `shell-storage-chip` / `ws-topstrip__settings-link` / `ws-topstrip__session-chip` / `ws-topstrip__session-chip-btn`** 全部改印章风:
- `position: relative` + `::before` 内容 `·`(archive-rose 14px,left 8-10px,absolute 居中)
- `border: 1px solid color-mix(in srgb, var(--archive-rose) 22%, var(--border))`(印泥粉红 22% 浓度边,V1 的 2 档 panel 之外,1 档 chip-only stamp 边)
- `background: transparent`(透明底,跟纸同色)
- `border-radius: 0`(档案册硬切角,删 V1 的 6px 圆角)
- `color: var(--text-primary)` / `font-weight: 500`(默认 500)
- hover:`border-color: color-mix(in srgb, var(--archive-rose) 40%, var(--border))` + `::before { color: var(--accent-rose); font-weight: 900 }`(墨点变实心,粉红加浓)
- `padding: 0 12px 0 18px`(左 18px 让出墨点空间)

### 4. mast paper-fiber 底(删 blur)

**`AppShell.vue` `.shell-mast`**:删 `backdrop-filter: blur(18px)` + 2-stop SaaS gradient + box-shadow;改纯 `linear-gradient(180deg, var(--bg-secondary) 96%, transparent, var(--surface-panel) 94%, transparent)` flat 底。

**`AppShell.vue` `.theme-kao .shell-mast` override**:删整段(min-height / padding / border-bottom / background / backdrop-filter / box-shadow)让 kao.css 接管。

**`kao.css` `.theme-kao .app-shell .shell-mast`**(V3 新增):扁平 archive-paper-soft 底 + 1px archive-gold 14% bottom edge + `position: relative` + no box-shadow。

**`kao.css` `.theme-kao .app-shell .shell-mast::before`**(V3 新增):paper-fiber speckle 3-stop radial-gradient(mix-blend-mode: multiply + opacity 0.32)— 复用 `.is-archive-paper::before` 同样的 3 层 radial-gradient(kao.css L109-122),让 mast 跟下面的页面是同一张纸。mast 内容 `> *` 加 `position: relative; z-index: 1` 浮在 speckle 之上。

### 5. tab 之间撕边虚线分隔

**`.shell-tabbar`**:`gap: 8px` → `gap: 0`(原来 V1 的 Material 间距)。

**`.shell-tab`**:`border: none; border-left: 1px dashed color-mix(in srgb, var(--border) 86%, transparent)`(tear-edge 撕边虚线,呼应 paper-tear 滤镜);**`.shell-tab:first-child`**: `border-left: none`(无 leading stub)。

**`.theme-kao .shell-tab`** override:`border-left-color: color-mix(in srgb, var(--archive-gold) 18%, transparent)`(kao 主题用 archive-gold,呼应档案册金色)。

**Mobile 760**:tab 仍横滚,撕边虚线在每 tab 左侧仍可见(无 indicator 影响)。

## 删 V2 indicator 原因

V2 commit `42ed7f3`(page-transition polish + tab indicator + caption fade)用 JS 算 active tab offsetLeft/offsetWidth,在底部滑 1 条 2px accent 渐变线 + box-shadow。这是 Material Design 3 + iOS segmented control 风格,跟档案册"印章 + 印泥"的墨色语言冲突:
- accent 是 SaaS 蓝色强调;档案册用 archive-rose 印泥色
- 2px 底部滑线是 affordance 语言("这里能切换"),档案册靠字重 bump + 印章 `◆` 表达
- JS 监听 resize + scroll + currentActivityKey + nextTick 维护 indicatorStyle,删后可省 ~25 行 + 一个 ref + 一个 `@media (prefers-reduced-motion)` 重写

印章 `◆` 是静态 ::before,无需 JS,无需 resize 监听,无 transition(refined-motion 友好),符合"删 SaaS 复杂度 + 加档案册签名"方向。
