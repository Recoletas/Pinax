## Summary

V2 commit `42ed7f3` 加的 page-route transition (enter 0.28s opacity + translateY 8px + scale 0.995,leave 0.16s translateY -6px) 在人眼阈值以下,切页像"没动"。V3 commit `cc12e62` 删了 V2 的 tab indicator 滑线、改用印章 `◆` 静态 `::before`,但印章只靠 class 切换显隐,无过渡 — 视觉上仍像瞬切。本轮(W1, V4)在 V3 基础上做两件事:

1. **页面切换加方向感**: enter 从横向位移进(向左 / 向右),leave 反向退出,人眼能识别"从哪来 → 到哪去"。
2. **tab 印章加微动效**: `◆` 从 translateY -4px + opacity 0 滑入到 translateY 0 + opacity 1,切走反向淡出,不再瞬切。

mast 路由标题 `caption-fade` 0.18s 已有,本轮不动(已 OK)。

## 方向感方案

### 决策依据

- V2 用 scale 0.995 + translateY 8px 给"页面软沉"感 — 但 scale 0.005 在 1080p 下 < 1px,人眼阈值之下;只有 opacity 起作用 = 纯淡入 = "页面切换没做"反馈。
- 加横向位移: 切到 activity index 更大 = 向右进;切到更小 = 向左进。enter 从 16px 起,leave 反向 10px 退,人眼能从方向感识别动作归属。
- 0 改 vue-router 路由配置,只动 `<transition>` 的 JS hooks(`@before-enter` / `@before-leave`)+ CSS `var(--page-direction)`。

### 实现

**`src/layouts/AppShell.vue` script 段**(L37-80,新加 44 行):
- `prevActivityKey = ref(currentActivityKey.value)` — 起始值锁住,避免 mount 时方向乱跳。
- `transitionDirection = ref(1)` — 默认 +1(向右进),首切不读为左滑。
- `watch(currentActivityKey, (nextKey, previousKey) => {...})` — 计算 direction:
  - 同 activity 子路由切换(例如 `/experience` ↔ `/opening`)→ direction = 0,只播 4px 纵向 settle,无横向位移(子路由不应假装是跨 section 移动)。
  - 新 index > 旧 index → direction = +1。
  - 新 index < 旧 index → direction = -1。
- `onPageBeforeEnter(el)` — 把 `+direction` 写进 `el.style.setProperty('--page-direction', ...)`,enter 元素读正号。
- `onPageBeforeLeave(el)` — 把 `-direction` 写进,leave 元素读反号 — enter 和 leave 镜像互推,而不是同向平移。

**`src/styles/main.css` page-rule 段**(L458-487,改 30 行):
- enter: `opacity 0 → 1` + `transform: translateX(calc(var(--page-direction, 1) * 16px)) translateY(4px) → 0`。
- leave: `opacity 1 → 0` + `transform: translateX(calc(var(--page-direction, 1) * 10px)) → 0`。
- duration: enter **0.32s**(原 0.28s),leave **0.18s**(原 0.16s)。总 0.50s 在 out-in 模式仍 < 0.55s 工位节奏,拖沓但能看清。
- easing: `cubic-bezier(0.22, 1, 0.36, 1)`(V2 沿用),跟 opening titleBreathe 同一根曲线。
- `scale(0.995)` 删(V2 的 0.005 形变 < 1px,徒增 transform 复杂度,本轮替换成有真实位移的 translateX)。
- `will-change: opacity, transform` 保留。

**`src/layouts/AppShell.vue` template 段**(L272-285,改 transition 标签):
- 加 `@before-enter="onPageBeforeEnter"` + `@before-leave="onPageBeforeLeave"`。
- `<transition>` 标签折 4 行让 hooks 可读,`name` / `mode` / 2 个事件各一行。

## 印章微动效

### 决策依据

V3 的 `.shell-tab::before` 在 inactive 时 `content: ""` 立刻切到 `::before` 伪元素上挂 content = "◆" — content 是非过渡属性,glyph 是 mount/unmount,0ms 切。

本轮把 inactive 的 `::before` 永远挂载(空 content),让 opacity + transform 走真过渡: glyph "从 -4px 落到 0" 看着是"印章盖下来",反向"印章抬起来"。width/margin-right 也加 transition 让字距同步收放,glyph 像有重量。

### 实现

**`src/layouts/AppShell.vue` scoped CSS .shell-tab::before 段**(L457-484,改 28 行):
- inactive (`:not(.active)` base):
  - `content: ""`(保持 V3-B 契约要求)
  - `opacity: 0`
  - `transform: translateY(-4px)` — 印章挂在 tab 上方,准备"盖下来"
  - `transition: opacity 0.2s, transform 0.2s, width 0.2s, margin-right 0.2s`(都用 `cubic-bezier(0.22, 1, 0.36, 1)`)
- active:
  - `content: "◆"`
  - `opacity: 1` + `transform: translateY(0)` + `width: auto` + `margin-right: 4px`
  - `font-size: 9px` + `color: color-mix(in srgb, var(--accent-rose) 82%, transparent)`(沿用 V3 印泥色)

**kao 主题 override 不动**(L913-915 仍只覆盖 `color: var(--archive-rose)`,transform/opacity 由 base rule 接管)。

## reduced-motion 守卫

### 现状

**`main.css` L522-538**(已有,本轮不动): `.page-route-enter-from { transform: none }` 已覆盖新 `translateX(... ) translateY(4px)`,`.page-route-leave-to { opacity: 0 }` 不动 transform 让 leave 自然位移 0;transition 在 enter-active / leave-active 上降到 0.01s ease。

**`AppShell.vue` scoped CSS 末尾新增 reduced-motion 块**(L1051-1059,新加 9 行):
```css
@media (prefers-reduced-motion: reduce) {
  .shell-tab::before {
    transition: opacity 0.01s ease, width 0.01s ease, margin-right 0.01s ease;
    transform: none;
  }
}
```
`opacity` 仍保留 0.01s 让 glyph 切换非闪烁,但 transform 取消(translateY -4px → 0 不再播);kao 主题的 archive-rose 颜色 override 仍生效,glyph 视觉到位。
