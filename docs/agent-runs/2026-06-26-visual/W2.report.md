## 1. 状态分层方案 (4 态 + 3 层)

### 1.1 `welcomeState` 状态机 (computed)

```
hasApiKey = false  → 'setup'        (未配 API key)
hasApiKey = true, hasWorldbooks = false  → 'choose-world'  (有 key 无世界)
hasApiKey + hasWorldbooks, hasSessions = false  → 'start'   (有世界无 session)
hasApiKey + hasWorldbooks + hasSessions = true  → 'resume' (有 session)
```

### 1.2 3 层入口 (改 v-for + computed, 0 静态 7 按钮)

| 层 | 来源 | 数量 | 变体 | 文案 (硬编码, 永远显示) |

| **Secondary** | `secondaryActions` computed, 永远显示 | 3 | `secondary` | **新世界** → /settings/worldbook, **写作** → /writing, **素材** → /materials |

## 2. 最近会话卡 (WelcomeView §"welcome-recent-session")

### 2.1 显示条件

`v-if="hasSessions && lastSession"` — 只在有 session 时显示,避免"无 session 还显示继续按钮"的空头承诺。

### 2.2 数据源

`lastSession` computed 从 `gameStore.sessions` 取最近 updatedAt 的 1 个,字段:

| 字段 | 来源 |
| --- | --- |
| id | session.id |
| title | session.title 或 "未命名会话" |
| updatedAt | session.updatedAt (sort 用) |
| messageCount | session.messages.length (fallback chatHistory.length) |
| worldbookId | session.worldbookId \|\| session.worldId |
| worldbookName | 从 worldStore.worldbooksIndex 找 id 对应 name |
| volumeLabel | 留空 (follow-up 字段, 当前 V2 不用) |

### 2.3 视觉 (V3 印章签名)

- 透明底 + `archive-rose 22%` 边 (跟 V3 stamp 语言一致)
- 前缀 `·` archive-rose 14px (墨点)
- 右上角 `进入` 按钮,hover 实心墨点
- a11y: `role="region"` + `aria-label="最近会话续接"` + 按钮 `aria-label` 插值 session title
- mobile 760 折叠: `position: static; width: auto; margin: 8px 16px 0;` (跟 command stack 一起 wrap)

### 2.4 路由消费

按 spec "如不支持就让 gameStore 论 currentSession" — `enterLastSession()` 调 `router.push('/experience')`,让 Experience.vue 的现有 mount 逻辑 (`gameStore.currentSessionId` 已在最近 session) 接管。0 改 router, 0 改 Experience.vue, 0 改 gameStore。

## 3. 当前冒险进度摘要 (跟最近会话卡合并)

按 spec "在最近会话卡里或旁边" — 我选**合并到最近会话卡** (1 个 card, 不堆叠):

## 4. Onboarding 完成态 (WelcomeView §"welcome-onboarding-done")

### 4.1 现状 (改前)

`v-if="isOnboarding"` 控制 onboarding 引导条 (3 步)。当 3 步全完成时 `isOnboarding = false`,引导条消失,主页**退化成纯入口墙** — 用户感觉"主页变空了"。

### 4.2 改后 (W2)

加 `v-else` 分支: 显示 1 行收尾印章 + 1 行小字:

```html
<div v-else class="welcome-onboarding-done" role="status" aria-label="已就绪">
  <span class="welcome-onboarding-done__check" aria-hidden="true">✓</span>
  <span class="welcome-onboarding-done__line">已就绪 · 可以开始</span>
</div>
```

- archive-olive 38% 边 + archive-olive 8% 背景 (跟 onboarding `is-done` step 颜色一致)
- ✓ glyph 用 archive-olive 90%
- 字号 11px + letter-spacing 0.16em + uppercase (跟 onboarding kicker 同)
- 视觉收尾: "完成 ≠ 消失,是换成已就绪印章"

## 5. featuredPreset 条件化 (移除静态依赖)

### 5.1 改前

```html
<aside v-if="featuredPreset" class="welcome-world-chip" aria-label="默认世界入口">
```

`featuredPreset` 是 `seedWorldbookPresets[0]` (边境王国), 永远显示。已完成用户看到 "默认世界入口" 不相关 (他们已经有自己的世界)。

### 5.2 改后 (W2)

```js
const showFeaturedPreset = computed(() => {
  return Boolean(featuredPreset.value) && !hasWorldbooks.value
})
```

```html
<aside v-if="showFeaturedPreset" class="welcome-world-chip" aria-label="默认世界入口">
```

- 只在 `!hasWorldbooks` 阶段显示 (即"刚配完 key 还没选世界"那 1 段时间)
- 配完世界后 chip 消失,主页不再"提示默认世界"

> 边境王国 seed (`featuredPreset`) 仍是 demo 主推, 但**只在用户需要引导时**显, 不再"已完成用户每天看到同样的默认世界 chip"。
