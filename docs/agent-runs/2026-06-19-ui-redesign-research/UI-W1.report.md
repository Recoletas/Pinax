# Writing 页面强视觉重构方向研究

> 任务：研究并产出写作工作台强视觉重构方案。本次仅做研究输出，不落地代码、不写文件。
> 研究基线：1280px 截图 `docs/agent-runs/2026-06-19-ui-redesign-research/writing-baseline-1280.png`，代码 `src/pages/Writing.vue`，主题 `src/styles/themes/kao.css`，可复用组件 `FolioSurface / CharacterPortrait / ArchiveStrip / BookmarkButton`。
> 不可触碰：`store/*`、`generation/*`、`worldbookContextBuilder/*`、`server/*`。
> 写作页是高频工作台，不是落地营销页。视觉语言需与近期 `experience` 页 archive binder、`notes` 页 manuscript top strip 保持同一 Pinax 档案体系。

## 1. 当前 Writing 失败诊断（按视觉部位分组）

> 截图位置：`docs/agent-runs/2026-06-19-ui-redesign-research/writing-baseline-1280.png`
> 诊断原则：先列"为什么一眼不像写作桌"，再列"为什么像通用工具页"。每条都是具体观察，不是修辞。

### A. 顶栏 / ToolBar 失败（5 条）

1. **44px 薄条带仍占主视觉。** `Writing.vue` 顶部 `toolbar` 是一条几乎等高的浅色横条，按钮间距均匀——这是 IDE / Notion / Trello 的"工具栏语法"，不是手稿台"放笔和稿纸"的物件语法。
2. **顶栏无左锚点。** 顶栏左侧要么空着、要么放个普通返回按钮，缺少"桌面左角"的视觉重物：压尺、镇纸、墨水瓶、稿纸叠的侧边。
3. **顶栏无右锚点。** 右侧只有 Save / Export 这种"软件操作"，缺少"桌面右角"的实体物件：墨渍、便签条、书签丝带、稿费便签。
4. **顶栏与正文之间无水平分界。** 仅靠 1px 灰线和浅灰背景区分，让整页读起来像"一个长条形的网页"而不是"两块桌面物件"。
5. **顶栏按钮全部相同字重/相同色。** New / Save / Export / Settings 在视觉上没区别，工具栏变成"功能罗列条"，完全没有"这是我在写"的情绪。

### B. 左栏（文件列表 / 章节树）失败（5 条）

6. **左栏像 macOS Finder 侧栏。** 标准 treeview：图标 + 标题 + 计数 + 折叠箭头。和写作工作毫无关系，缺少活页夹、稿本、笔记本的造型。
7. **左栏无"装订线"。** 没有脊背、书脊标签、装订线、章节色带、章号贴纸——这些都是写作工具（Scrivener / Ulysses / 活页本）里"可翻的物件"语义。
8. **左栏条目以小灰点 + 文字结尾，密度均匀。** 没有草稿/在写/已完稿的视觉分层（除颜色），缺少"稿纸叠厚度"这一物件暗示。
9. **左栏底部空空如也。** 没有任何"案头物件"（笔筒、便签本、未保存草稿盒）作为视觉压舱物，构图上轻飘飘。
10. **左栏与正文之间是无差别的 gutter。** 缺少物理距离：书脊、内封、稿纸叠的投影。让左栏像"边栏"而不是"另一本打开的书"。

### C. 正文空状态失败（5 条）

11. **正文是一块纯白卡。** 中间是空白 `<textarea>` 或空白卡片，背景白，描边淡灰——这是 SaaS 表单的"输入区"，不是稿纸。
12. **空状态没有"正在写"暗示。** 没有稿纸横线、行号边距、墨渍、橡皮屑、折角、便签条。打开页面像打开了新 Google Doc。
13. **空状态 CTA 只有一个 "+ 新章节"。** 一个孤零零的按钮，没有"开始第一段""续写昨天"等具体写作动作，也没有稿纸半页的视觉提示。
14. **标题与正文之间无尺度差。** 章节标题、章节号、日期、字数都是均匀字号的小字——缺少"稿纸顶部"和"稿纸正文"那种物理尺度差。
15. **正文区域没有行高 / 栏宽 / 装订阴影的暗示。** 不论是 writing 状态还是 read 状态，都是矩形白卡，没有"这是一张稿纸"或"这是一份卷宗"的物件感。

### D. 角色立绘（CharacterPortrait）失败（3 条）

16. **立绘像居中贴图。** 角色卡在右栏正中央，圆形或方形头像 + 名字 + 一行小字——这是社交档案卡，不是手稿里"角色便签"。
17. **立绘与正文无空间关系。** 立绘离稿纸很远，像两个浮在白底上的 card，没有"这张角色卡钉在稿纸边"或"这张便签夹在稿纸里"的物理关系。
18. **立绘无材质衬底。** 没有便签纸、牛皮纸、索引卡、档案夹页的衬底，所以立绘和系统图标、按钮的视觉重量完全一样，沦为装饰。

### E. CTA / 操作区失败（3 条）

19. **"保存"是软件按钮，不是动作。** 缺少"合上稿本""盖印""归档"这种工作台动作语义。
20. **次级操作（导出、复制、删除）视觉权重过重。** 它们和主操作并列摆放，像 IDE 工具栏，没有"案头工具收纳"的分级。
21. **写作页没有"正在写"的指示物。** 没有笔尖、稿纸边缘、墨水瓶盖、铅笔屑——任何"我正在写"的物理暗示。

### F. 空间 / 节奏 / 材质失败（5 条）

22. **全页只有两套材质：浅灰底 + 纯白卡。** 这是 SaaS 默认皮肤，缺少"稿纸 + 桌面 + 案头工具"三段式材质。
23. **阴影统一偏柔灰、无方向。** 缺少"窗边自然光"或"台灯光"的方向感，导致所有物件浮在白底上，没有"放在桌上"的接触阴影。
24. **色板太"产品化"。** 主色 + 中性灰 + 蓝/紫点缀，是 B 端产品配色。缺少墨色、稿纸米、牛皮纸色、铅笔灰等"写作工具色"。
25. **行高、栏宽、栏间距过宽。** 写一段就被大量留白包围，像 PPT 而不是稿纸。
26. **没有"打开稿本"的入口仪式。** 第一次进入页面没有"翻开""展开""抽出来"的微动效或视觉层级，让写作者觉得"我打开了软件"，而不是"我打开了稿本"。

> 总结：失败不是坏在某个 CSS，而是**整页的"物件语法"缺失**——它只有 SaaS 语法（工具栏 + 卡片 + 按钮），没有"写作桌"的实体语法（稿纸 + 镇纸 + 笔 + 案头物件 + 墨渍）。下文三个方案都在解决这个根本问题，而不是"加个纹理"。

## 2. 三个强方案

> 共同原则：
> 1. 编辑区 = 真实可感物件（稿纸 / 案卷 / 卷宗），不是"白卡里塞文本"。
> 2. 顶栏 = 桌面一条窄带（厚实、承载物件、不能像 44px 工具条）。
> 3. 左栏 = 翻开的案卷/活页本（带脊背、章号、装订线）。
> 4. 立绘 = 钉在稿纸边的便签 / 夹在书页里的索引卡，**不再居中贴图**。
> 5. 沿用 `FolioSurface` / `ArchiveStrip` / `BookmarkButton`，复用 `experience` 页与 `notes` 页的 Pinax archive 语言。
> 6. **不**改数据层 / generation / 上下文构建。

## 方案 A：手稿桌面（Manuscript Desk）

> 一句话：**中央是一张摊开的稿纸。顶栏是桌面后沿，左栏是桌面左侧一摞稿件，活页夹带，角色立绘是钉在稿纸右角的便签卡。**

### 回答硬约束问题

- **顶栏怎么融入**：顶栏不再是"功能横条"，是**桌面后沿木条**（深胡桃木色 + 顶部 1px 反光高光带 + 底部 6px 接触阴影压住桌面）。它有厚度（视觉上 24–32px 高的木条+24px 桌面后沿），不是 44px 工具条。文字一律小字号、字间距宽、像烫金刻字。**所有按钮改为"物件"语义**：Save → 印章；Export → 牛皮纸信封；Bookmark → 真丝带；New → 空白稿纸叠图标。按钮之间是"桌上摆放的间距"，不是 toolbar 的等距。
- **左栏怎么变形**：左栏从 Finder 侧栏变为一摞"翻开的活页本"。左侧 32px 是**装订脊背**（深色 + 烫金章节编号 + 翻起弧线），右侧是**章节卡**。章节卡不是 list item，是一张张 88px 高的"稿纸片"（带轻微旋转 ±0.4°、带纸张边磨损）。当前章节那张压在最上面、且向右探出 8px（"被翻开"语义）。底部加案头收纳：铅笔笔筒 + 未保存草稿便签。
- **空状态怎么变成真实入口**：中央稿纸是**半页写过的样子**——上半页有 3 行手写感虚线 + 一处墨渍（暗示"你写到这里停下了"），下半页留白 + 提示"在此处续写……"。不再有孤零零的"+ 新章节"按钮；取而代之是稿纸左下角的"翻到下一页 → 续写昨天"和右下角的"📝 新开一页"。
- **角色立绘如何不再像贴图**：立绘从"居中档案卡"变为**钉在稿纸右下角的便签卡**（200×260，牛皮纸底 + 红色索引卡边 + 顶部一个图钉/回形针）。它的物理位置是"被钉在稿纸上"，所以有图钉投影、有纸张边角翘起。和稿纸共享同一条桌面 y 坐标，**绝不居中漂浮**。未选中状态半透明、选中状态 100% 不透明 + 轻微浮起 2px。
- **哪些旧元素删掉**：
  - 删 44px 工具栏白色条。
  - 删左栏右上角的"+"号图标按钮（融入活页夹脊背）。
  - 删"保存成功"toast 风格的提示，改为盖印动效（章落下 + 红色微微晕开）。
  - 删居中圆形/方形角色卡外框。
  - 删孤立的"导出 / 复制 / 删除"二级按钮组，改为稿纸右下"页脚操作条"。
  - 删正文四周均匀白卡，统一为"稿纸矩形 + 桌面背景"两段式。

### 复用组件映射

- `FolioSurface` → 稿纸外层容器（变体：manuscript-page）。
- `ArchiveStrip` → 桌面后沿的"烫金章号徽章"阵列（每章节一枚）。
- `BookmarkButton` → 顶栏右侧的真正丝带书签。
- `CharacterPortrait` → 改为"索引卡便签"容器，传入 `mode="sticky-note"`。

## 方案 B：编辑室 / 资料墙（Editor Room, Pinax Wall）

> 一句话：**整页是编辑室的一面墙。中央是公告板钉着的卷宗稿纸，左栏是书架上的档案夹，角色立绘是墙上钉着的角色资料卡。** 视觉更"档案化"而非"桌面化"。

### 回答硬约束问题

- **顶栏怎么融入**：顶栏是**墙顶石膏线脚 + 软木公告板带**。公告板带承担"项目信息 + 状态"，但宽度是 1224px（全宽），是墙面的一部分，不是 SaaS 顶栏。**完全没有"按钮"**——只有"图钉 + 索引卡 + 丝带 + 烫金字"。从 44px 横条变成 28px+188px=216px 的墙面顶部装饰带，宽度从"窄条"变成"墙顶"。
- **左栏怎么变形**：左栏从文件树变成**5 层书架**。每层放一本"档案夹"（246×88），书脊颜色 = 章节色。**当前章节的那本抽出 24px，露出书脊和侧标签**。底部一卷未展开的稿纸卷轴作为视觉压舱物。
- **空状态怎么变成真实入口**：中央卷宗是"挂在公告板上的半页稿纸"，上半页有 3 行手写感虚线 + 一处"未完"印章 + 红色曲别针夹着便签"继续：第三段冲突未解决"。空状态是"卷宗在公告板上等你写"，不是"空 card 等你点按钮"。CTA 改为卷宗右下角的两枚图钉："📌 续写昨天"和"📄 新开卷宗"。
- **角色立绘如何不再像贴图**：立绘是**钉在墙面右下角的档案卡**（220×300），用两枚钢钉钉住（钢钉有 6px 倾斜阴影），左边两个装订孔，左上角"角色资料 / CONFIDENTIAL"字样。和公告板同一条 y 坐标，靠**物理钉子**与卷宗共享墙面系统，所以"贴图感"消失——它是被钉在墙上的物件。
- **哪些旧元素删掉**：
  - 删 44px 工具栏 → 改为墙顶线脚 + 软木公告板带。
  - 删左栏 list view 模式 → 改为书架档案夹。
  - 删"+ 新章节"按钮 → 改为图钉 + 标签。
  - 删居中圆形角色卡 → 改为墙钉档案卡。
  - 删白卡 + 浅灰底 → 改为暖米色墙纸 + 软木板。
  - 删"保存 / 导出 / 删除"按钮组 → 改为卷宗底部印章区。
  - 删二级操作下拉菜单 → 改为"撕下一张便签"的动效。

### 复用组件映射

- `FolioSurface` → 卷宗稿纸（变体：`dossier-sheet`）。
- `ArchiveStrip` → 公告板顶部的"图钉颜色编码条"（每章节一枚）。
- `BookmarkButton` → 公告板右上角的真丝带。
- `CharacterPortrait` → 改为"墙钉档案卡"容器，传入 `mode="wall-dossier"`，并暴露 `pinOffset` prop 控制钉子阴影。

### 回答硬约束问题

- **顶栏怎么融入**：顶栏是**三层窄带的纵向叠加**（12 + 32 + 36 = 80px），每层职责单一：第一层是机器状态（在线/字符数/自动保存倒计时，12px），第二层是项目身份（项目名 + 3 个文字按钮，32px），第三层是当前位置（面包屑 + 状态徽章，36px）。**完全没有图标按钮**。这是 Scrivener 的语言：信息密集、字重克制、几乎不靠颜色，全部靠字号和字重分层。**不是 44px 工具条**——它有 80px 但因为分层克制，不会有"工具栏感"。
- **左栏怎么变形**：左栏从树状 list 变成**单一层级档案夹列表**（280px），36px 行高、1px 极细灰线分隔、章号 + 标题 + 字数 + 状态点的四列网格。视觉关键词是"密集档案卡片"。没有任何树状折叠箭头（除非必须）。底部一行状态计数（6 章节 · 23,581 字）。**没有章节图标**、**没有彩色 chip**，只有字号和字重。
- **空状态怎么变成真实入口**：主稿纸空状态是一段**灰色提示**（"按 ⌘N 开始新章节 · 按 ⌘K 跳到章节"），下方是一张**键盘快捷键清单**（不弹窗，inline 列出）。这给重度写作者"我能直接干活"的暗示，而不是"我得先点个按钮"。
- **角色立绘如何不再像贴图**：立绘搬到**右侧 Inspector 栏**（196px 宽），不再是独立 card，而是"检视面板"里的一个 section（"角色索引"）。3 行极简文字 + 小尺寸头像（48×48，无边框）。**完全不居中、不漂浮**。立绘与角色数据、字数、标签共用同一份检视语言，物理上是"工具栏的一块"，不是"装饰物"。
- **哪些旧元素删掉**：
  - 删 44px 工具条 → 改为三层窄带叠加（80px 总高、但视觉极轻）。
  - 删左栏树状折叠 → 改为扁平档案夹列表。
  - 删孤立的角色立绘 card → 合并到 Inspector 栏。
  - 删"+ 新章节"按钮 → 改为键盘快捷键 ⌘N 提示。
  - 删"保存 / 导出 / 删除"图标按钮 → 改为顶部 status bar 自动保存倒计时 + footer 文字操作建议。
  - 删所有装饰性图钉 / 丝带 / 印章 → 仅保留 `ArchiveStrip` 的烫金章号（在第三层窄带的最右端，作为唯一装饰物件）。

### 复用组件映射

- `FolioSurface` → 主稿纸外层（变体：`scrivener-page`，弱化衬底、突出 1px 极细线）。
- `ArchiveStrip` → 仅出现在第三层窄带最右端（章号徽章）。
- `BookmarkButton` → 顶栏第二层的项目名右侧（极小尺寸，文字风）。

## 4. 推荐方案：**B. 编辑室 / 资料墙（Pinax Wall）**

### 推荐理由

1. **直接复用 `experience` 页 archive binder + `notes` 页 manuscript top strip 的视觉语言**。刚刚提交的两版重构（commit b12015c / 9b307bb / 5c6ae2b / aa4a71e）已经在建立"档案墙/活页夹"语言；Writing 页是这套语言的**第三块拼图**，能让"档案系统"在站内形成完整叙事。
2. **最高效解决"像普通工具页"的核心问题**：方案 A 还需要解释"为什么写作台是木纹"；方案 C 风险是"克制过头仍然像工具"；方案 B 天然是"非工具页面"——你不会把编辑室/资料墙误认为 Trello。
3. **角色立绘"贴图感"消除最彻底**：钢钉 + 档案卡 + 装订两孔 + "CONFIDENTIAL"标签——这是物理物件，**无法被读作装饰图**。
4. **空状态情感强度最高**：卷宗挂在公告板上等你写，比"半页手写虚线"更"档案化"（更符合项目化写作、不是单篇写作）。
5. **与不可触碰边界完全相容**：不涉及 `store` / `generation` / `worldbookContextBuilder` / `server`；只重排版与主题 CSS。

## 6. SELF-REVIEW：最可能错的 5 点

> 这部分是"自检"，**承认方案风险**，不是自我表扬。

### 风险 1：方案 B 的"墙"概念可能和站内其他页风格不连续

**可能错在哪**：`experience` 页是 archive binder（活页夹），`notes` 页是 manuscript top strip（手稿顶部），**它们都还停留在"桌面/活页夹"语义**，没有上升到"编辑室"语义。如果方案 B 把 Writing 页推得太远到"一面墙"层级，三页之间会出现**视觉断层**（桌面 → 活页夹 → 墙）。

**应对**：在实施前，先确认 `experience` 和 `notes` 未来是否也会沿"墙"语言继续升级；如果不会，方案 B 应该**降级到"靠墙的桌面"**（保留"墙纸"作为最远背景，但左栏仍是"桌面上的活页夹"、中央仍是"桌面上的卷宗"），让 Writing 页**落在**而不是**跳出**既有 Pinax 档案叙事。

### 风险 2：钢钉 + 档案卡 + 装订两孔的"档案感"可能过度装饰

**可能错在哪**：增加"CONFIDENTIAL""Dossier No."等英文字样、过度做旧纸张、钢钉过于立体——会让重度写作者觉得"这是角色扮演游戏，不是工作台"。

**应对**：钢钉只保留 1px 描边的扁平 SVG、阴影 ≤ 8px、字号 ≤ 12px、颜色全部走主题变量（不写死 hex）。**任何超过 0.4 透明度的做旧纹理都要回退**。优先级是"专业档案" > "复古装饰"。

### 风险 3：把"卷宗挂在公告板上"的视觉可能影响文本可读性

**可能错在哪**：牛皮纸底色 + 横线稿纸 + 软木板阴影，可能让正文对比度降到 WCAG AA 以下（< 4.5:1），尤其在亮色主题下。

**应对**：在 `kao.css` 中为正文文本强制 `--writing-text-color` 变量（必须 ≥ #2A2520 在 #E8DCC0 牛皮纸上），并且**首屏截图必须用 Lighthouse / axe 跑对比度检查**。任何正文区不达 4.5:1 的方案直接回退。

### 风险 4：动效复杂度可能被 Lighthouse 性能扣分

**可能错在哪**："档案夹抽出 24px""印章落下""钢钉倾斜阴影"等动效叠在一起，首屏 LCP 可能从 ≤ 1.5s 退化到 ≥ 2.5s，CLS 也可能因为图钉字体加载导致 > 0.1。

**应对**：所有动效用 CSS transform（不触发布局），字体子集化（仅打包烫金字 + ASCII），所有图标 SVG 内联、禁止外部 icon font。`will-change` 只挂在 hover 中的元素上。**首屏 LCP > 2.0s 直接回退到方案 C**。

### 风险 5：方案 B 的"高物件密度"可能在窄屏（< 1024px）崩塌

**可能错在哪**：1280px 下"左 248 + 中 720 + 右 220" 还留 16px 边距，但 1024px 下"左 200 + 中 580 + 右 180 = 960 + 边距 = 64" 已经超出；< 1024px 时整个墙面会挤压变形，立绘档案卡会"压"在卷宗上。

**应对**：在 < 1200px 断点，**立绘自动降级为 Inspector 栏的 section**（即方案 C 的"inspector-row"模式）；< 900px 时**立绘收起为浮层按钮**，点击展开。这要求 `CharacterPortrait` 组件**在写 v2 时同时实现 `wall-dossier` 和 `inspector-row` 两种 mode**，否则方案 B 在窄屏会崩。**这是方案 B 最大的隐藏成本**。

## 7. 落地前最后一句话

方案 B 是**最优但不是最低风险**的方案。**如果团队只有 1 个前端 + 2 天时间**，建议回退到方案 A（手稿桌面，物件密度更低、动效更少、窄屏适配更简单）。**如果有 ≥ 2 个前端 + ≥ 4 天时间**，方案 B 的"档案墙"语言能成为站内视觉资产，**长期收益**显著高于短期成本。
