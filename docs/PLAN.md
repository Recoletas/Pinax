# 叙事工具链总计划（UI 可用性重构版）

> 当前默认只看本文件和 [plan/current-execution-plan.md](./plan/current-execution-plan.md)。

## 1. 当前阶段目标

当前主目标不是继续扩功能，而是先完成 UI 可用性重构。兼容性只作为回归底线，不作为保留旧入口、旧布局或旧交互路径的理由。

1. 让用户能在最短路径内完成体验、世界书、写作、素材和分镜的核心任务。
2. 减少重复入口和互相遮挡的浮层，降低首次理解成本。
3. 统一页面骨架和高频操作位置，让跨页面迁移不需要重新学习。
4. 在 UI 基线稳定前，暂停新增重功能入口。

## 2. 已完成能力（归档摘要）

以下能力已具备，文档只保留结果摘要，不再展开实现细节：

- 会话与运行态边界已稳定。
- 世界书导入与高级设置双入口已落地。
- 素材中台与写作消费链路已打通。
- 生成调用已收口到统一任务层。
- 编辑器与统一智能顾问已收口。
- 记忆系统已达到可用基线（细节优化暂缓）。
- 统一分镜对象和版本保存入口已启动并接入主路径。

## 3. 当前核心问题（UI 维度）

### 3.1 可用性问题

- 高频操作入口分散（顶部按钮、侧栏、悬浮按钮、弹层内二次按钮），用户不知道哪个才是主路径。
- 同类动作文案和位置在页面间不一致，用户从体验切到写作、设定或分镜时需要重新判断。
- 复杂面板默认信息密度偏高，新用户首次进入负担较重。
- 旧能力入口仍保留过多，导致“能找到功能”不等于“知道该怎么完成任务”。

### 3.2 兼容与布局风险

- 多页面存在 `height: 100vh + overflow: hidden + 多个 fixed 浮层` 的叠加模式，移动端和低分辨率下容易出现遮挡、滚动锁死、层级冲突。
- 体验页和写作页都存在多个右下/右侧固定入口，交互热区重叠风险高。
- 页面级布局断点策略不一致，导致同一用户在不同模块获得不同交互习惯。

## 4. 重构原则

1. 用户可用性优先于保留旧 UI 兼容；如果旧入口造成理解负担，可以合并、降级或移除。
2. 先统一交互骨架，再优化局部视觉。
3. 先保证核心路径能完成，再追求精细样式。
4. 单页面只保留一个主操作区和一个辅助浮层入口。
5. 统一断点与弹层策略：桌面、平板、手机三档行为一致。
6. 已稳定功能不重写底层逻辑，只做 UI 壳层与交互路径收口。

决策顺序：

```text
任务可完成 > 入口清晰 > 页面一致 > 移动端不阻塞 > 旧入口兼容 > 视觉细节
```

## 5. 本阶段交付定义

达到以下条件才算 UI 重构阶段完成：

1. 新用户能完成五条核心路径：开始体验、保存/整理素材、写作消费素材、导入或编辑世界书、保存或导出分镜。
2. 每个高频任务在 3 步内可达（进入、编辑/生成、确认）。
3. 体验、写作、世界书导入三大高频页面在 360px、768px、1280px 下无关键遮挡和不可点击区域。
4. 关键浮层（速记、素材收件箱、顾问、分镜导出提示）不存在层级互相遮挡与背景滚动失控。
5. 顶部导航、侧栏、按钮分层在各页面保持一致的交互语义。

## 6. 非协商规则

- 不新增第二套状态源。
- 不绕开统一生成任务层。
- 不在页面层直接读写持久化。
- 不在同一页面继续增加新的悬浮入口。
- 不为了兼容旧入口而牺牲当前主路径清晰度。
- 每个 UI 改动必须给出明确的用户可感知变化和回归结果。
