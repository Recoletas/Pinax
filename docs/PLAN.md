# 小说体验框架 - 开发计划

> 本文档规划"小说体验"模块的开发方向：基于世界书驱动、AI辅助的写作体验工具。用户通过导入世界书或小说，模拟小说情节推进，辅助创作。
>
> 文档导航：请先阅读 `docs/README.md` 获取分区说明与推荐阅读路径。

---

## 一、核心定位

### 1.1 这是什么

| 对比维度 | 酒馆 (SillyTavern) | 本框架 (小说体验) |
|---------|-------------------|------------------|
| **核心场景** | 与AI角色对话/角色扮演 | 模拟小说情节、写作辅助 |
| **用户角色** | 扮演故事中的角色 | 可以扮演角色，也可以旁观 |
| **叙事方式** | 第一人称对话为主 | 第三人称叙事 + 第一人称对话混合 |
| **输出目标** | 有趣的对话体验 | 可沉淀到写作的叙事片段 |
| **卡片系统** | 角色卡 = AI扮演的角色 | 角色卡 = 世界观参考，可导入/导出 |
| **机制展示** | 所有功能常驻显示 | 平时简洁，事件触发时才显示对应机制 |

**一句话定义：** 这是一个以世界书为驱动、叙事为核心、写作辅助为导向的工具。扮演与机制并存，但平时保持简洁，事件触发时才展现对应功能。

### 1.2 核心交互模式：事件驱动机制

```
┌─────────────────────────────────────────────────────────┐
│  正常状态：简洁界面                                      │
│  - 主体验区：叙事输入/输出                              │
│  - 侧边栏：角色、位置、活动记录（可折叠）              │
│  - 战斗/任务/交易等机制 → 隐藏                          │
└─────────────────────────────────────────────────────────┘
                          │
            ┌─────────────┴─────────────┐
            │                           │
            ▼                           ▼
┌───────────────────────┐   ┌───────────────────────┐
│  触发战斗事件          │   │  触发交易事件          │
│  → 显示战斗面板        │   │  → 显示交易面板        │
│  → 回合制操作         │   │  → 物品/金币交换      │
│  → 完成后自动隐藏     │   │  → 完成后自动隐藏     │
└───────────────────────┘   └───────────────────────┘
```

**意义：** 不预先堆砌功能，只在需要时呈现。用户不会因为看到"背包"按钮但里面没东西而困惑。

### 1.3 当前系统问题

**数据流断裂：**
- `gameStore.inventory` 和 `gameStore.quests` 存在但从未被UI组件使用
- `StatusBar.vue` 管理的 `writing_character` 与 `gameStore.playerCharacter` 不同步
- `Inventory.vue` 实际管理"场景"数据但名字误导
- `QuestLog.vue` 实际管理"活动时间线"，不是任务系统
- `WorldMap.vue` 的场景只有 `{id, name}`，与 `Inventory.vue` 的场景数据不同步

**酒馆痕迹过重：**
- 当前界面直接暴露了战斗、交易、随机遭遇等关键词触发逻辑
- 但UI没有配套的展示机制（事件触发时没有对应的交互面板）
- 机制存在但没有正确的呈现时机

**世界书系统缺失：**
- 现有的 `server/data/worlds/` 是硬编码临时数据
- 没有世界编辑器，无法从SillyTavern导入

## 二、UI设计原则

### 2.1 核心原则

```
简单优先 → 详情其次 → 高级设置折叠
```

**主界面原则：**
- 只展示创作当下最需要的东西
- 侧边栏精简，必要时可折叠
- 世界书、编辑器、设置等不常用的放到二级页面或抽屉/模态框
- 不要把"可能性"当成"功能"堆在界面上

**界面层级：**

| 层级 | 内容 | 访问方式 |
|------|------|---------|
| **主体验区** | 当前场景、输入框、叙事输出 | 默认显示 |
| **侧边栏** | 当前角色、当前位置、最近活动 | 默认显示，可折叠 |
| **世界书选择** | 选择/切换世界书 | 顶部下拉拉或点击弹出 |
| **编辑器** | 世界书条目编辑 | 新页面或大模态框 |
| **高级设置** | AI参数、注入规则、导入导出 | 设置页或详情页 |

## 三、数据模型

### 3.1 核心数据结构

```
WorldBook (世界书)
├── id: string
├── name: string
├── description: string
├── author: string
├── version: string
├── createdAt: timestamp
├── updatedAt: timestamp
├── settings: { scanDepth, tokenBudget, recursiveScanning }
└── entries: WorldEntry[]

WorldEntry (世界条目)
├── id: string
├── keys: string[]           # 触发关键词
├── keysSecondary: string[]  # 次要关键词
├── content: string          # 条目核心内容
├── type: EntryType          # location | character | item | lore | quest | event | general
├── name: string             # 条目名称（用于显示）
├── injection: InjectionConfig
│   ├── mode: 'selective' | 'constant' | 'group'
│   ├── probability: number (0-100)
│   ├── cooldown: number
│   ├── depth: number
│   ├── excludeRecursion: boolean
│   └── group: string | null
├── relations: {
    tags: string[],
    locations: string[],
    characters: string[],
    events: string[]
  }
└── metadata: { createdAt, updatedAt, importSource }

Character (角色卡)  # 与SillyTavern兼容
├── id: string
├── name: string
├── description: string      # 角色描述（对应SillyTavern的description）
├── personality: string[]    # 性格标签
├── greeting: string | null
├── avatar: string | null
├── worldEntryId: string | null  # 关联的世界书条目
└── extensions: object       # 扩展字段（兼容SillyTavern格式）

ExperienceState (体验状态)
├── worldbookId: string | null
├── characterId: string | null
├── locationId: string | null
├── sceneId: string | null
├── mood: number (0-100)
├── timeProgress: number
├── flags: Record<string, any>
├── activities: Activity[]
├── activeMechanism: MechanismType | null  # 当前激活的机制：'combat' | 'trade' | 'quest' | 'dialogue' | null
└── mechanismState: MechanismState | null   # 当前机制的内部状态

MechanismState (机制状态，根据activeMechanism不同而不同)
├── combat: { enemy, playerHP, enemyHP, turns, choices }
├── trade: { npcId, items, gold, mode: 'buy' | 'sell' }
├── quest: { questId, objectives, progress }
└── dialogue: { characterId, history }

Activity (活动记录)
├── id: string
├── title: string
├── type: 'event' | 'milestone' | 'decision' | 'encounter'
├── createdAt: timestamp
└── relations: { type, targetId }[]
```

### 3.2 与SillyTavern的兼容性

**角色卡兼容：** 直接复用SillyTavern的角色卡JSON格式，可以导入导出。

**世界书兼容：** 完整解析SillyTavern的 `entries` 结构，字段一一映射。

| SillyTavern 字段 | 内部字段 | 备注 |
|-----------------|---------|------|
| `entries.{id}.key` | `keys` | 一致 |
| `entries.{id}.keysecondary` | `keysSecondary` | 一致 |
| `entries.{id}.content` | `content` | 一致 |
| `entries.{id}.comment` | `name` | 映射到name字段 |
| `entries.{id}.selective` | `injection.mode='selective'` | 布尔→枚举 |
| `entries.{id}.constant` | `injection.mode='constant'` | 布尔→枚举 |
| `entries.{id}.group` | `injection.group` | 一致 |
| `entries.{id}.depth` | `injection.depth` | 一致 |
| `entries.{id}.probability` | `injection.probability` | 一致 |
| `entries.{id}.cooldown` | `injection.cooldown` | 一致 |
| `entries.{id}.excludeRecursion` | `injection.excludeRecursion` | 一致 |

## 四、功能模块

### 模块 1：世界书编辑器（详情页）

**定位：** 核心功能，但非常用功能。通过独立页面访问。

#### 1.1 功能列表

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 条目列表 | 搜索、筛选（类型/分组）、排序 | P0 |
| 条目编辑 | key、name、content、type、injection | P0 |
| SillyTavern导入 | 上传JSON，直接解析导入 | P0 |
| SillyTavern导出 | 导出为兼容JSON | P0 |
| 小说AI导入 | 上传小说文本，AI提取实体生成世界书 | P1 |
| 分组管理 | 创建/编辑/删除分组 | P1 |
| 批量编辑 | 多选条目，批量修改 | P1 |
| 预览调试 | 输入测试文本，查看触发条目 | P2 |
| 版本历史 | 自动保存历史，支持回滚 | P2 |

#### 1.2 SillyTavern导入流程

```
1. 上传 .json 文件（或拖拽）
2. 解析 entries 结构
3. 字段映射（SillyTavern → 内部格式）
4. 显示预览：条目数量、类型分布、关键词统计
5. 用户确认 → 写入存储
6. 完成，跳转世界书详情页
```

#### 1.3 小说AI导入流程

```
1. 上传小说文本（TXT/MD/EPUB）
2. 自动分段（按章节或固定长度）
3. 对每段调用AI提取：
   - 地点 → 创建 location 条目
   - 角色 → 创建 character 条目 + 角色卡
   - 物品/道具 → 创建 item 条目
   - 重要事件 → 创建 event 条目
   - Lore/设定 → 创建 lore 条目
4. 生成关系映射（角色-地点、事件-角色等）
5. 用户预览，确认后正式导入
6. 导入后可编辑补充
```

### 模块 2：体验主界面（首页）

**定位：** 始终可见，是用户的主要操作界面。保持简洁。

#### 2.1 功能列表

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 世界书选择 | 顶部下拉切换当前世界书 | P0 |
| 叙事输入 | 文本框输入动作/指令 | P0 |
| 叙事输出 | 显示AI生成的叙事内容 | P0 |
| 角色显示 | 侧边栏显示当前角色 | P0 |
| 位置显示 | 侧边栏显示当前位置 | P0 |
| 最近活动 | 侧边栏显示最近3-5条活动 | P0 |
| AI开关 | 切换AI开/关 | P0 |
| 快捷输入 | 场景/对话/心理/继续 快捷按钮 | P1 |
| 上下文可视化 | 调试模式下显示已注入的条目 | P2 |

#### 2.2 不在主界面的功能（移到详情页）

以下功能通过二级入口访问，不在主界面直接展示：
- 世界书编辑（进入编辑器页面）
- 角色卡管理（进入角色页面）
- AI参数设置（设置页面）
- 导入/导出（设置或编辑器页面）
- Token预算（设置页面或hover显示）

#### 2.3 事件触发面板（平时隐藏，触发时弹出）

机制存在但平时不显示，只在叙事触发相关事件时才弹出：

| 触发场景 | 显示面板 | 面板内容 |
|---------|---------|---------|
| 战斗中 | 战斗面板 | 回合制操作：攻击/防御/技能/道具/逃跑 |
| 交易时 | 交易面板 | 物品列表、金币交换 |
| 任务接取 | 任务面板 | 任务描述、目标、奖励 |
| 遭遇NPC | 对话面板 | 角色对话、历史关系 |
| 探索发现 | 发现面板 | 地点描述、出口选项 |

**面板交互流程：**
```
1. AI叙事触发战斗/交易/任务等事件
2. 叙事区域出现选项按钮（如"应战"/"逃跑"）
3. 用户选择后，对应面板从底部/侧边滑入
4. 用户完成交互（战斗结束/交易完成）
5. 面板收起，继续普通叙事模式
```

**定位：** 辅助功能。角色卡是创作者的参考资料，不是AI扮演的角色。

#### 3.1 与世界书的关系

```
角色卡 ←关联→ 世界书条目
  │                  │
  └── name, description, personality
                         │
                         └── keys, content (角色背景设定)
```

#### 3.2 功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 角色卡导入 | 从SillyTavern格式导入角色卡 | P0 |
| 角色卡导出 | 导出为SillyTavern兼容格式 | P0 |
| 角色卡列表 | 管理已导入的角色卡 | P0 |
| 关联世界书 | 角色卡关联到世界书中的角色条目 | P1 |
| 角色关系 | 从活动记录中自动追踪角色间关系 | P2 |

### 模块 4：AI叙事系统

**定位：** 核心能力。决定叙事质量。

#### 4.1 分层提示词架构

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: 系统提示词 (System Prompt)                  │
│ - 叙事者身份定义                                    │
│ - 叙事格式规则 (*动作* "对话")                      │
│ - 风格指导（文学性/简洁/网文风）                    │
└─────────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────┐
│ Layer 2: 世界书注入 (WorldBook Injection)            │
│ - 当前位置/角色相关条目（关键词匹配）               │
│ - 常量条目（constant=true）                         │
│ - 分组条目（按group权重选择）                      │
└─────────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────┐
│ Layer 3: 情境上下文 (Situation Context)              │
│ - 当前角色状态（心情数值）                          │
│ - 时间/年代背景                                     │
│ - 最近活动摘要                                     │
│ - 情绪氛围引导                                     │
└─────────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────┐
│ Layer 4: Few-shot 示例 (可选)                       │
│ - 2-3个高质量叙事片段                               │
│ - 根据场景类型（战斗/日常/感情）动态选择           │
└─────────────────────────────────────────────────────┘
```

#### 4.2 叙事风格预设

用户在设置中选择预设，影响系统提示词的语气和格式：

| 预设 | 特点 | 适用场景 |
|------|------|---------|
| 文学性强 | 注重文笔、环境描写、内心刻画 | 严肃向小说、历史 |
| 流畅叙事 | 节奏紧凑、可读性强 | 商业向小说、网文 |
| 对话丰富 | 人物对话推动情节 | 都市、校园 |
| 简洁白描 | 少修饰、重情节 | 快节奏、悬疑 |

#### 4.3 上下文压缩策略

```
当前问题：简单截断最近N条消息，丢失重要设定

改进方案：
1. 标记重要信息：角色关系变化、剧情转折、关键发现
2. 优先保留：上述重要信息 + 最近消息
3. 中间部分：AI摘要压缩
4. 压缩触发：token接近预算上限时
```

### 模块 5：创作顾问

**定位：** 辅助功能。被动响应，不主动打扰创作流程。

#### 5.1 功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 问答模式 | 用户主动提问，获取建议 | P0 |
| 叙事分析 | 分析最近一段叙事的优缺点 | P1 |

| 世界书补充 | 发现设定缺失时建议添加条目 | P2 |

## 八、与酒馆的关键区别（提醒）

### 8.1 哪些保持一致

- **角色卡格式**：完全兼容SillyTavern的JSON格式，可以互相导入导出
- **世界书格式**：完整兼容 `entries` 结构，字段一一映射
- **AI接口**：保持现有的多Provider支持（OpenAI/Claude/DeepSeek等）
- **扮演功能**：酒馆的角色扮演能力本框架也有

### 8.2 核心差异

| 差异点 | 酒馆 | 本框架 |
|--------|------|--------|
| **界面风格** | 功能常驻（背包/任务/战斗按钮始终显示） | 平时简洁，事件触发才显示对应机制 |
| **叙事方式** | 第一人称对话为主 | 第三人称叙事 + 第一人称对话混合 |
| **输出目标** | 娱乐对话 | 可用于写作的叙事片段 |
| **侧边栏** | 游戏状态（HP/金币/背包）始终可见 | 创作参考（角色/位置/活动），可折叠 |

### 8.3 本框架独有

- **事件驱动机制**：战斗/任务/交易等平时隐藏，叙事触发时才弹出对应面板
- **世界书编辑器**：可视化编辑世界书条目
- **小说导入**：从小说文本自动生成世界书
- **叙事风格预设**：文学性/流畅/对话/简洁 4档
- **叙事片段导出**：可将精彩段落导出到笔记

## 十、数据存储

### 10.1 localStorage

| Key | 内容 | 备注 |
|-----|------|------|
| `worldbooks_index` | 世界书列表元数据 | ~1KB/世界书 |
| `worldbook_{id}` | 单个世界书完整数据 | ~100KB-1MB |
| `experience_state` | 当前体验状态 | ~5KB |
| `characters` | 角色卡列表 | ~5KB/角色 |
| `experience_settings` | AI和叙事风格设置 | ~2KB |

### 10.2 存储策略

- 世界书数据大，使用独立key存储（`worldbook_{id}`）
- `worldbooks_index` 只存元数据列表（id、name、entryCount）
- 按需加载：读取世界书时只加载索引，需要详情才加载完整数据
- 大型世界书（>500条目）考虑IndexedDB
