# 叙事工具链总计划

> 后续推进默认只看本文件和 [plan/current-execution-plan.md](./plan/current-execution-plan.md)。专题文件只在需要实现细节、边界定义或验收口径时打开。

## 1. 当前判断

项目已经不适合继续按“诗歌页 + 散文页 + 体验页”这种功能并列方式理解。

现在真实已经落地的，是一套逐步成形的叙事创作工具链：

```text
设定 / 世界书 / 角色 / 约束
	-> 体验生成与探索
	-> 素材沉淀与整理
	-> 写作加工与续写
	-> 候选设定 / 记忆确认
	-> 分镜化与导出
```

因此后续规划必须遵守两条总原则：

1. 诗歌和散文不再作为产品中心继续扩张，而是作为工具链中的专用生成入口或加工模式。
2. 剩余工作不再是“补几个页面功能”，而是把长期资产层和结构化输出层收口完成。

## 3. 工具链中的核心对象

后续所有新功能都必须能明确挂到这些对象之一，不能再以页面私有结构充当事实来源。

| 对象 | 含义 | 当前状态 |
|------|------|----------|
| Session | 一次体验会话的容器，负责恢复、切换、保存 | 已稳定 |
| RuntimeState | 会话内运行态，如消息、地图、活动、角色、阶段状态 | 已稳定 |
| NarrativeAsset | 内容流转的中间资产，承接体验、素材、章节和候选设定 | 已使用 |
| WritingProject / Chapter | 正式写作载体，消费素材并产出正文 | 已使用 |
| WorldbookEntry / WorldbookDraft | 正式设定与待确认设定 | 已使用 |
| MemoryCandidate / MemoryRecord | 待确认记忆与已确认长期记忆 | 已使用，待收尾 |
| GenerationTask | 所有 AI 调用的统一任务抽象 | 已使用 |
| StoryboardDocument / StoryboardVersion | 统一分镜输出对象 | 已启动，编导组合函数、散文导出、诗歌导出、体验和章节草稿都已接入，主入口导出已统一到校验保存入口 |
| AdvisorTask / AdvisorResult | 统一智能顾问任务和结构化结果 | 已落地 |

## 4. 模块重新定义

后续文档、导航和实现都应按下面的含义对齐：

| 模块 | 在工具链中的角色 |
|------|--------------------|
| 体验 | 负责探索、互动、生成剧情和收集原始材料 |
| 素材 | 负责沉淀、筛选、转流向，是工具链的交换层 |
| 写作 | 负责章节加工、续写、改写和结构组织 |
| 设定 | 负责世界书、角色设定、约束和回写边界 |
| 记忆 | 负责长期上下文候选与确认，不直接替代世界书 |
| 分镜 | 负责结构化输出、版本化和导出 |
| 顾问 | 负责跨阶段审阅、建议和辅助分析 |
| 诗歌 / 散文 | 负责提供特定风格文本与分镜生成入口，不再作为独立产品中心 |

## 5. 当前剩余主任务

后续主要推进三条主线：

1. 统一分镜输出层。
这是当前优先级最高的整体优化块，需要把诗歌、散文、体验片段和章节片段统一映射到同一套 storyboard schema、版本和导出校验。当前已启动统一 `StoryboardDocument` 和版本 service，主入口导出也已经统一到校验保存入口，后续继续迁移旧页面私有分镜入口。

2. 产品叙事与信息架构收口。
即使底层已经像工具链，导航和页面命名仍带着旧的功能页叙事，需要继续把“诗歌/散文”降级为模式或入口，把“素材/设定/分镜”抬为主线能力。当前一级导航和首页入口已经开始切到体验、写作、设定、分镜。

3. 记忆系统细节收尾。
记忆已经能候选、确认、分 scope、同步和读取，也补上了同步状态可见、批量整理、冲突和去重基础能力；排序筛选细化和更强语义去重先不作为当前主线。

编辑器与统一智能顾问这条线已完成，见 [plan/editor-advisor-unification.md](./plan/editor-advisor-unification.md) 和 [LOG.md](./LOG.md).

## 7. 非协商规则

- 不新增第二套体验状态源，短期仍以主体验 store 为准。
- 不新增页面层直接调用 `sendChat()` / `sendChatStream()`。
- 不新增页面层直接调用 `runGenerationRetryPlan()`，优先抽专用 service，再走 `runGenerationTask()` / `runGenerationStreamTask()`。
- 不新增页面层直接读写 `localStorage`，统一走 service 或 store。
- 不自动写入世界书和长期记忆，必须先经过候选或显式确认。
- 不在诗歌、散文、体验页内部各自长出独立分镜 schema。
- 所有用户可感知改动都必须能说明“用户在界面上做什么会看到变化”。
