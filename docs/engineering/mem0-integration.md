# 记忆力系统 - mem0 集成

> mem0 API 已接通，尚未融合进项目主逻辑。本文档记录集成规划。

## 1. mem0 简介

mem0 是一个长期记忆服务，核心能力：
- **跨会话记忆**：记住用户偏好、创作风格、常用设定
- **实体关系**：自动维护角色/地点/事件之间的关系图
- **自适应**：根据使用情况自动优化记忆权重

## 2. API 状态

mem0 服务已部署，API 端点可用：
- `POST /v1/memories` - 存储记忆
- `GET /v1/memories` - 检索记忆
- `DELETE /v1/memories/{id}` - 删除记忆

## 3. 集成计划

### 3.1 第一阶段：写作上下文记忆

```
用户创作时 → 自动提取关键设定 → 存入 mem0 → 后续创作时自动注入
```

| 记忆类型 | 示例 | 注入场景 |
|---------|------|---------|
| 角色设定 | 主角名、性格、背景 | AI 续写/扩展时 |
| 世界观 | 地点名称、势力关系 | 世界书生成时 |
| 写作偏好 | 文风、叙事视角、章节长度 | 所有 AI 生成时 |

### 3.2 第二阶段：体验状态记忆

```
用户体验时 → 记录关键决策 → 存入 mem0 → 后续体验时参考
```

| 记忆类型 | 示例 | 注入场景 |
|---------|------|---------|
| 决策历史 | 玩家做出的重要选择 | AI 叙事时 |
| 关系变化 | 角色间关系演变 | 对话/互动时 |
| 剧情进度 | 当前章节/时间线 | 自动注入上下文 |

## 4. 数据流设计

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   用户创作   │ ──► │  mem0 API   │ ──► │  记忆存储    │
└─────────────┘     └─────────────┘     └─────────────┘
                          │
                          ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  AI 生成时   │ ◄── │  记忆检索    │ ◄── │  上下文组装  │
└─────────────┘     └─────────────┘     └─────────────┘
```

## 5. 实现路径

| 阶段 | 任务 | 优先级 |
|------|------|--------|
| P0 | mem0 API 客户端封装 | API 已通，封装后接入 |
