# 小迭代记录

> 局部语义变化、小修、小调查的记录载体。
> **当 [`../../LOG.md`](https://github.com/Recoletas/Pinax/blob/main/docs/LOG.md) 容纳不下、但又不需要进 RFC 时**，用它。

## 何时使用

- 一个修复合并了，但修过的代码区域有什么坑值得留一句话给将来的人？
- 一项局部调查给出了非显然的结论？
- 一次小重构被推到 N 个文件，想让"为什么"可被回溯？

**不要**用于：完整功能上线（直接更新 `LOG.md`），或大方向变更（用 RFC）。

## 格式

每个小迭代一个 markdown 文件，命名 `YYYY-MM-DD-<short-slug>.md`：

```markdown
# YYYY-MM-DD: <一句话标题>

- **问题**: <一两句话>
- **方案**: <做了什么>
- **验证**: <哪些测例 / 命令 / 视觉验收>
- **局部影响**: <改了哪些 surface、是否影响调用方>
- **未解决 / 风险**: <留下来让将来看到的事项>
```
