# 运维与故障排查

## 1. 常见问题清单

### 1.1 设置弹窗打开后页面异常
- 现象：设置页无响应，控制台出现空值读取错误。
- 优先检查：`useApiSettings` 是否存在默认配置对象与安全 merge。
- 处理建议：保证配置读取失败时仍返回可渲染的默认值。

### 1.2 架构守卫失败（直接调用 sendChat）
- 现象：`npm run verify` 失败，报业务层直接调用 `sendChat`。
- 优先检查：`pages/stores/composables` 是否存在直接导入。
- 处理建议：改用 `runGenerationRetryPlan`，补齐重试与解析逻辑。

### 1.3 文件上传在自动化环境下 ENOENT
- 现象：浏览器自动化上传本地文件时报路径不存在。
- 场景特征：Linux 侧路径无法直接映射到浏览器侧盘符。
- 处理建议：将文件镜像到浏览器可访问路径后再上传，避免跨环境路径直接复用。

## 2. 排查流程模板

1. 先复现：记录页面、输入、操作序列。
2. 再分层：确认是 UI 状态、Store 动作、API 通信还是持久化问题。
3. 最后收口：补最小修复并执行 `npm run verify`。

## 3. 发布前冒烟建议

- 普通入口：预设导入、文本导入、AI 生成各走一轮。
- 冲突策略：至少验证一次“覆盖同名”。
- 高级设置：验证分组增删改迁移与导出。
- 主体验页：验证世界书切换后剧情生成链路可用。
