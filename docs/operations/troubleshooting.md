# 运维与故障排查

> 当前 UI 阶段优先判断“用户是否能完成任务”。兼容问题只在造成遮挡、不可点击、滚动锁死或输入不可用时升为高优先级。

## 1. 常见问题清单

### 1.1 设置弹窗打开后页面异常
- 现象：设置页无响应，控制台出现空值读取错误。
- 优先检查：`useApiSettings` 是否存在默认配置对象与安全 merge。
- 处理建议：保证配置读取失败时仍返回可渲染的默认值。

### 1.2 架构守卫失败（直接调用 sendChat）
- 现象：`npm run verify` 失败，报业务层直接调用 `sendChat`。
- 优先检查：`pages/stores/composables` 是否存在直接导入。
- 处理建议：改用对应的专用 `services/*`，再由 `runGenerationTask` / `runGenerationStreamTask` 发起生成，补齐重试与解析逻辑放在 service 层。

### 1.3 文件上传在自动化环境下 ENOENT
- 现象：浏览器自动化上传本地文件时报路径不存在。
- 场景特征：Linux 侧路径无法直接映射到浏览器侧盘符。
- 处理建议：将文件镜像到浏览器可访问路径后再上传，避免跨环境路径直接复用。

### 1.4 移动端输入时界面被顶出或遮挡
- 现象：输入法弹出后编辑区不可见，或页面出现异常跳动。
- 优先检查：页面是否依赖 `height: 100vh` 且同时存在固定头部/底部。
- 处理建议：统一视口高度策略，减少多层滚动容器并验证软键盘场景。

### 1.5 浮层打不开、遮挡或无法关闭
- 现象：弹窗被其他悬浮按钮覆盖，或关闭后背景仍不可滚动。
- 优先检查：同页固定定位元素数量、z-index 层级、背景滚动锁逻辑。
- 处理建议：收口悬浮入口数量，统一弹层层级与打开/关闭流程。

### 1.6 页面出现横向滚动或按钮超出可视区
- 现象：小屏设备出现横向滚动条，关键按钮被裁剪。
- 优先检查：固定宽度、`min-width`、大尺寸弹层宽度计算。
- 处理建议：按断点重设宽度策略，确保 360px 场景可操作。

## 2. 排查流程模板

1. 先复现：记录页面、输入、操作序列。
2. 先判断任务是否被阻断：能否继续编辑、保存、关闭、确认或返回。
3. 再分层：确认是 UI 状态、Store 动作、API 通信还是持久化问题。
4. UI 问题额外执行：360 / 768 / 1280 三档回归 + 亮暗主题回归。
5. 最后收口：补最小修复并执行 `npm run verify`。

## 3. 发布前冒烟建议

- 普通入口：预设导入、文本导入、AI 生成各走一轮。
- 冲突策略：至少验证一次“覆盖同名”。
- 高级设置：验证分组增删改迁移与导出。
- 主体验页：验证世界书切换后剧情生成链路可用。
- UI 基线：验证固定入口、弹层层级、滚动行为在移动端可用。
