# 核心概念

项目里有一些自创的术语。第一次看会混。读完这一节就清楚了。

## 世界书（WorldBook）

**世界书**就是你这个故事的"设定大字典"——地理、势力、人物关系、特殊规则、关键历史事件……全都拆成一条条"条目"存着。

进入体验页冒险或写作时，AI 并不是每次都把整本世界书读一遍。系统在每轮生成时会扫描最近上下文里的关键词，匹配到的条目才会被注入到 prompt 里，所以条目写得好不好直接决定 AI 写得准不准。

> 详细操作流程（怎么导入、怎么分组、怎么处理冲突）看 [世界书工作流指南](../guides/worldbook-workflow.md)。

## 条目（Entry）

世界书里的最小单位。一条典型的条目长这样：

| 字段 | 是什么 |
| --- | --- |
| `name` | 条目名，比如"蜀山派" |
| `type` | 分类：地点 / 角色 / 势力 / 事件 / 物品 / 规则 |
| `keys` | 触发关键词，比如 `["蜀山", "蜀山派", "剑宗"]` |
| `content` | 条目正文，AI 看到关键词命中时会把这段读进去 |
| `mode` | `constant`（永远注入） 或 `selective`（按 keys 命中才注入） |
| `group` | 所属分组，便于批量管理 |

## 结构化设定（StructuredSettings）

世界书是"自由文本"——AI 自己读、自己理解。**结构化设定**是"字段化"的：朝代表、角色卡、地点卡、场景卡都是一个个带字段的对象。

字段化的好处是 UI 上能直接画表格、改下拉菜单，AI 也能按字段精确定位信息。坏处是新增类型要改代码。

世界书和结构化设定是互补的：

- **世界书**管"原则、规则、风格、世界观"——这些适合自由文本
- **结构化设定**管"具体的人、具体的地方、具体的时间线"——这些适合字段

## 角色（Character）

每个角色有一张卡：

- `name` / `description`（外貌、背景）
- `personality`（性格、说话方式）
- `greeting`（开场合词）
- `example_dialogue`（几条对话样本，AI 会模仿语气）

主角、AI 主持的 NPC、对话里的配角都共用这张卡的结构。

## 状态（Status）

体验页右边的状态栏那一排数字：

| 状态 | 一般含义 |
| --- | --- |
| `vitality` | 体力 / 生命值 |
| `mood` | 心情 / 情绪 |
| `money` | 金钱 |
| `level` / `exp` | 等级和经验 |

具体字段名是项目里写死的，但实际数字会跟着剧情推进变化。

## 时间（Time）与朝代（Era）

两套时间并行：

- **日常时间**：`day`（日期序号）+ `period`（早晨 / 上午 / 中午 / 下午 / 傍晚 / 夜晚 / 深夜）
- **历法时间**：`eraId`（朝代 ID）+ `year` + `month` + `day`

仙侠、奇幻这种题材常用朝代。都市、科幻可以用现代公历。AI 写剧情时如果搞混了时间，状态栏的颜色会变红提示。

## 里程碑（Milestone）

剧情推进到关键节点时弹出的提示卡：

- `location-unlock` —— 解锁了一个新地点
- `character-appearance` —— 新角色登场
- `time-skip` —— 时间跃迁（比如一章结束后跳到第二天）

可以点"接受"写入状态栏，也可以点"忽略"维持原样。

## 机制（Mechanism）

体验页右下角那一排可点的小按钮，触发 AI 主持特定类型的互动：

- `combat` —— 战斗
- `trade` —— 交易
- `quest` —— 接 / 交任务
- `dialogue` —— 对话

点哪个，AI 就按哪个模式来写。

## 叙事素材（Narrative Asset）

写作或冒险过程中产生的“金句 / 重要事件 / 关键描写”，可以单独拎出来存成素材卡。素材卡带情绪标签（悲 / 喜 / 怒 / 惊 ……），在画布里能按情绪、关系和时间顺序继续整理。

`localStorage` 键名：`narrative_assets_v1`。

## 记忆候选（Memory Candidate）

AI 写作时会自动识别哪些信息"值得长期记住"——比如"玩家不喜欢冗长描述"或"作者偏好用短句"——这些会先进 **记忆候选池**。

池里的条目你可以：

- **采纳** —— 写入 Mem0，成为跨会话的偏好
- **忽略** —— 留在本地，下次 AI 不再提示
- **丢弃** —— 删掉

`localStorage` 键名：`memory_candidates_v1`。

## Mem0 偏好记忆

跨会话记住的写作风格 / 决策偏好。跟记忆候选的区别：

- **记忆候选** —— 本次会话发现的可记的事，等你点头
- **Mem0 偏好** —— 已经点头确认的，跨会话、跨项目都生效

要开 Mem0 得自己注册一个 [mem0.ai](https://mem0.ai) 的账号拿到 API key，然后在 **设置 → 记忆系统** 里填。详见 [04-configuration.md](./04-configuration.md)。

## 顾问（Advisor）

AI 之外的第二意见。点章节旁的 **顾问** 按钮，会调 `/api/openclaw/proactive-check` 对当前内容做一次体检，返回三类提示：

- `critical` —— 严重问题（逻辑冲突、设定违反）
- `warning` —— 值得回头看看（节奏、人物一致性）
- `info` —— 可选建议（润色、扩展）

要顾问工作需要配 OpenClaw 网关 token，详见 [04-configuration.md](./04-configuration.md)。

## 分镜（Storyboard）

把一章小说拆成一组“镜头”，每个镜头是一段场景描述 + 镜头语言（远 / 中 / 近、特写、推拉摇移）。在“画布”里能可视化排时间轴。

`localStorage` 键名：`storyboard_documents_v1` / `storyboard_snapshots_v1`。

## 卡片画布（ProseEssay Canvas）

节点-边-时间轴三件套。每个节点是一张卡（可以是素材、镜头、章节片段），边是节点之间的关系（前置 / 后继 / 引用 / 镜像）。支持 commit / branch / pile（堆叠）三种操作。

`localStorage` 键名：`prose_cards_v1` / `prose_edges_v1` / `prose_outline_v1` / `prose_timeline_v1` / `prose_piles_v1` / `prose_commits_v1` / `prose_branches_v1`。
