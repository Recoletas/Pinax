# Pinax 首页视觉改版设计

状态：待实现

> 本文只描述当前首页 `src/views/WelcomeView.vue` 的视觉与布局改版。路由、业务数据流、工作台页面和世界书 / 顾问专题都不在本次范围内。

## 1. 目标

把当前首页从“功能入口页”收敛成一个更像 Pinax 的工作台门面：

- 首屏第一眼读到 `Pinax`，而不是旧产品语境。
- 视觉语言靠近 VSCode 一类的 IDE 首页：克制、密度适中、层次清楚。
- 保留现有入口分发能力，但让入口更整齐、更安静、更像工作台。
- 不做营销式 hero，不做大面积装饰，不做新导航体系。

## 2. 范围

### 2.1 允许改动

- `src/views/WelcomeView.vue`
- 必要时的少量 `src/styles/main.css` 通用 token 补充

### 2.2 不改内容

- 不改 `src/router/index.js`
- 不改体验、世界书、写作、素材、画布页面逻辑
- 不改顾问专项文件
- 历史残留 `src/pages/Home.vue` 已在确认无路由引用后清理；当前首页事实文件只有 `src/views/WelcomeView.vue`

## 3. 页面结构

首页采用“IDE-heavy 的双区结构”：

1. 顶部轻量 chrome
   - 左侧显示 `Pinax`
   - 右侧保留主题切换
   - 整体高度压低，像应用窗口标题栏，不像营销页头部

2. 中央主区
   - 左侧为品牌与状态区
   - 右侧为入口区
   - desktop 上形成明显的两栏关系
   - 平板和手机上收敛为单栏顺序排列

3. 底部状态栏
   - 维持 IDE 式的细状态条
   - 只承载轻量环境信息和当前状态，不放解释性长文

## 4. 视觉语言

### 4.1 品牌区

- `Pinax` 作为首屏主字样，字体尺寸明显高于其他文本，但不做夸张 hero。
- 副标题只保留一句短描述，例如“叙事创作工作台”。
- 品牌区旁边可放 2-3 个 compact chips，表达当前工作流的核心能力。

### 4.2 入口区

- 入口不再像“功能宣传卡”，而是更像 command entry。
- 每个入口保持统一行高、统一图标尺寸、统一描述长度。
- 主入口优先级明显高于其他入口，但不使用夸张视觉按钮。
- hover 只做轻微边框、背景和位移变化。

### 4.3 色彩与材质

- 使用深浅中性的背景层次，避免高饱和渐变主导整个首页。
- 以边框、分割线、低对比底色建立层次。
- accent 只用于当前焦点、主入口和少量强调态。
- 视觉密度参考 VSCode，但不要复制其具体品牌细节。

## 5. 内容模型

首页保留现有入口分组，但重排呈现方式：

- `开始` 分组：体验、写作
- `工具` 分组：世界书、素材、画布

分组标题更短，说明文字更收敛，避免两层以上解释。

首页文案统一使用 `Pinax` 语境，清理掉任何仍然出现在首屏的旧产品名。

## 6. 交互规则

- 点击入口仍然直接进入对应工作区。
- 主题切换保留。
- 不新增额外弹窗、侧栏、引导页或轮播区域。
- 不增加复杂首屏动画，只保留轻微淡入和 hover 节奏。
- 键盘和鼠标的可达性保持不退化。

## 7. 响应式规则

- 1280px：两栏结构完整展开，品牌区与入口区分层清晰。
- 768px：改为单栏，但仍保持模块顺序和间距节奏。
- 360px：入口卡片变成单列堆叠，文字不挤压、不裁断。
- 状态栏在窄屏下收缩为最核心信息。

## 8. 验收标准

1. 首屏一眼读到 `Pinax`，且首页语气统一。
2. 视觉上更像 IDE 起始页，而不是普通入口列表。
3. 入口仍然清晰可点，但整体更协调、更安静。
4. 360px、768px、1280px 三档下都不出现横向滚动或拥挤断裂。
5. 不影响现有工作台页面、路由和业务逻辑。

## 9. 实施备注

- 首选在 `src/views/WelcomeView.vue` 内完成结构和样式收敛。
- 如需共享 token，只做最小量的 `main.css` 补充，不扩散到其他页面。
- 不需要引入新依赖，不需要改 router，不需要新状态源。
- 如果后续需要新增实验性门面，单独开文件和路由，不要再复活 `src/pages/Home.vue` 这种未挂路由的并行入口。
